# 原供应链平台框架保留说明

本 skill 继承 `$supply-chain-platform-growth` 的判断框架。变化只发生在执行机制：从“一次性按 checklist 汇总”，改成“多角色研究 + 中间文件 + 反方审查 + 投资人写作”。

## 核心判断句

```text
产业周期够大 + 供应链位置关键 + 订单出货领先 + 利润平台换台阶 + 当前估值未过度透支
```

这句话不能被改写成普通的“景气度高、业绩增长、估值合理”。它包含五个递进问题：

- 下游产业周期是否真实，并且仍有未来 1 到 3 年可见度。
- 公司在供应链中是否处于关键位置，而不是普通代工或可替代配套。
- 订单、排产、出货是否领先财报，并能传导到收入和利润。
- 公司是否从旧利润平台切到新利润平台。
- 当前市值是否已经提前定价未来 2 到 3 年利润。

## 分析链条

正式分析按这条链推进：

```text
产业大趋势
-> 客户资本开支
-> 供应链地位
-> 订单/排产/出货
-> 收入确认
-> 利润中枢上移
-> 市场身份重估
-> 股价重估或估值透支
```

供应链平台股不是单纯靠“技术垄断”涨，也不是简单靠“代工扩量”涨。它的核心是：在产业周期爆发时，市场重新定价公司的大规模交付能力和供应链组织能力。

## 六层结构

### 第一层：产业周期是否足够大

问：下游需求是否真实爆发，并且仍有未来 1 到 3 年可见度？

看客户资本开支、新产品平台、价值量、复杂度、ASP、供给扩张、库存和价格压力。没有产业周期，供应链平台重估很难持续。

### 第二层：市场身份是否切换

问：市场是否正在把公司从旧身份切到新身份？

常见路径：

```text
消费电子/通信设备/普通制造
-> AI 服务器/高速网络/算力硬件供应链平台
```

身份切换必须由订单、出货、利润率、客户结构或市场定价行为共同验证。不要把“AI 概念”直接等同于“身份切换成功”。

### 第三层：供应链地位是否关键

问：公司是核心平台、关键供应商、重要配套，还是普通供应商？

看客户认证、复杂产品量产、良率、全球产能、交付规模、供应链组织能力、多产品多客户扩散、替代难度、替代周期和议价权。供应链平台公司的壁垒不一定是芯片 IP 或品牌心智，也可以来自认证、规模、交付、良率、成本和全球组织能力。

### 第四层：订单出货是否领先财报

问：增长是否已经在订单、排产、出货中出现，并且能传导到收入和利润？

看明确订单、量产、出货放量、客户项目、海关/产业链数据、排产、收入确认节奏、毛利率、净利率、存货、应收、合同负债和经营现金流。

订单信号分级：

| 信号 | 含义 |
| --- | --- |
| 强信号 | 明确订单金额/数量、量产、出货放量、财报利润验证 |
| 中信号 | 扩产、设备、客户认证、合作框架、产能爬坡 |
| 弱信号 | 概念、传闻、笼统进入供应链、未量化表述 |

### 第五层：利润中枢是否上移

问：公司是否从旧利润平台切到新利润平台？

季度利润年化只能作为压力测试：

```text
Q4 x 4 不是精确预测，而是检验公司是否进入新利润平台的情景假设。
```

利润桥：

```text
未来利润 =
旧平台利润
+ 新周期新增利润
- 扩张费用/折旧/财务成本/爬坡损耗
```

必须回答高利润季度是偶发还是新平台起点，下一年订单能否支撑全年高利润，收入增长是否带来毛利率、净利率或规模效应改善，现金流、应收、存货是否支持利润质量。

### 第六层：估值是否透支

问：当前股价反映的是哪一年、哪一种利润情景？

必须用当前市值反推 2025、2026E、2027E、2028E PE，并说明利润假设属于保守、中性还是乐观。如果只有乐观情景才能消化估值，不能写成便宜。

## 常见误判

- 把行业景气误判成公司壁垒。
- 把扩产误判成订单。
- 把订单误判成利润。
- 把收入增长误判成利润中枢永久上移。
- 把季度高利润误判成全年常态。
- 把 AI 概念误判成市场身份切换成功。
- 把供应链关键地位拔高成顶级科技垄断。
- 忽略应收、存货和现金流恶化。
- 忽略客户压价、竞争扩产和估值锚回落。

## 报告必须回答的问题

- 公司是否符合供应链平台型框架，还是只是一家普通制造/代工/配套公司。
- 市场正在交易什么：身份切换、订单放量、利润中枢上移、估值透支，还是错杀修复。
- 产业周期是否仍在上行，客户 capex 是否能支撑未来 1 到 3 年。
- 公司在供应链中处于什么位置，壁垒和可替代性如何。
- 订单出货是否真实放量，财报是否已确认，增长质量是否可靠。
- 利润平台是否换台阶，未来利润桥是否能走通。
- 当前市值反映到哪一年利润，是否已经透支。
- 哪些证据会推翻主线逻辑。
