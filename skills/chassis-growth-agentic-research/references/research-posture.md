# Research Posture

本文件承接主控的研究姿态、市场变量写法、标题判断和证据分级时机。主控在写市场重定价主线、模块文件、提纲、扩写蓝图和终稿前必须读取本文件；本文件不替代 `modules/*.md`，而是约束所有模块如何保持进攻性和边界感。

## Core Posture

正式报告要同时有进攻性和边界感。先把市场正在重新定价的变量讲完整，再用证据等级和反方审查约束它；不要先把变量降噪成低弹性结论。反方审查是刹车和仪表盘，不是方向盘：它只负责告诉主线在什么条件下降级、证伪或需要降低置信度，不负责把开篇改成“机会存在但待验证”的两头话。

- 公告事实确定底线：收入、利润、现金流、客户、产能、研发和管理层披露。
- 市场口径打开弹性：第二曲线空间、客户导入、产品放量、份额、ASP/毛利率、平台身份、核心资源、组织投入。市场口径不等于事实，但经常是股价和利润斜率正在交易的真正变量，必须进入主线、竞争/客户验证链、利润桥、敏感性和跟踪体系。
- 情景假设负责天花板：用底线/基准/上行三层情景说明“如果市场口径成立，利润中枢怎样抬高；如果不成立，回到哪一档”。
- 中期进攻 QA 负责防止报告缺章、过薄和低进攻性：在提纲前检查市场变量、竞争客户链、承接动作、上行情景、利润斜率和跟踪体系是否都进入正文逻辑。它不是风控闸门，不得把高重要市场口径从正文删回证据台账。
- 反方审查负责证伪和事实边界，不负责把终稿改成“证据不足所以全部降级”的保守综述。反方输出必须同时保留乐观派押注、悲观派怀疑、验证信号和降级路径。
- `investment_logic_card` 负责固定主线魂魄：最大预期差、核心分歧、最短因果链、稀缺资源或关键瓶颈、最小利润模型、第一驱动参数、数量乘数、利润中枢区间、置信度、验证点、一票否决证伪点和 200-350 字“投资人首页判断”。后续模块可以校准它，但不能绕过它另起一个首鼠两端的主线。

## Single Spine And Sharpness Rule

成长股研究要锋利，必须先把“第一性赚钱变量”挑出来。正式研究只允许一条第一主线，其他线索只能扮演三种角色：增强主线、校准主线、证伪主线。不要把所有模块平铺成同等重要的章节。

所有模块和终稿段落都必须回答：

```text
这条信息改变最大预期差吗？
它影响最小利润模型的哪个参数：单位经济、数量乘数、兑现节奏、份额、ASP、毛利率、费用率、折旧/财务成本、税率、少数股东、现金回收？
它是第一驱动、第二驱动、第三驱动，还是只是背景？
如果它不成立，利润模型哪个参数下修，主线降到哪一档？
它应该进入主段落、表格、跟踪清单、反方章节，还是删除？
```

写作时优先保留“能改变利润模型参数”的材料。不能改变参数的事实，即使真实，也只能作为背景；不能改变最大预期差的长段行业介绍，即使正确，也应压缩或删除。锋利不是强行乐观，而是让每个判断都能落到同一条因果链上：

```text
最大预期差
-> 稀缺资源/关键瓶颈
-> 公司为什么拿到
-> 第一驱动参数如何变化
-> 单位经济 × 数量乘数 × 兑现节奏
-> 利润中枢区间
-> 验证/证伪
```

## No Teaching Mode Rule

正式报告不是课堂讲义。术语解释、行业常识和研究框架只能作为读者跨过门槛的最短台阶，不能成为正文主菜。

段落保留前必须问：

```text
这段是否包含公司特定事实、数字、客户/订单/产能/竞争证据？
这段是否改变或约束最小利润模型参数？
这段是否让第一主线增强、校准或证伪？
这段是否给出验证或证伪指标？
```

如果四个答案都是“否”，删除。若只有“帮助理解背景”，压缩成一句并贴到公司事实后面。不要写“我们先理解某概念”“一般而言某行业如何”“投资者需要关注某变量”这类教学语气；直接写该变量在本公司利润桥里改变什么。

## Direction First, Skeptic Later Rule

终稿前段必须先给读者一个清楚的研究方向。正确顺序是：

```text
核心交易逻辑
-> 市场误定价
-> 最大预期差/核心分歧
-> 行业瓶颈/稀缺资源
-> 公司为什么能拿到这块利润
-> 第一驱动参数
-> 最小利润模型与利润中枢区间
-> 已验证证据与待验证变量
-> 反方审查、降级路径和跟踪指标
```

错误顺序是：

```text
合同/订单/新业务存在
-> 但是证据不足
-> 仍需验证
-> 后续关注
```

前 30% 篇幅不得让“待验证、证据不足、现金流压力、治理折价、风险提示、反方观点”成为章节中心。可以写置信度和证据等级，但必须先写清楚：若核心变量成立，收入、毛利率、费用率、现金流或归母利润会被怎样改写。若报告读完只剩“有机会但要等财报确认”，说明主线写作失败。

## Minimum Profit Model Rule

成长股研究必须给出可争论的数字靶子。利润桥不能只写方向，必须尽量拆到最小经营单元：

```text
单位收入
- 单位可变成本
- 单位运维/服务/制造成本
- 折旧/摊销
- 财务成本
- 税
= 单位归母净利或经营利润

单位利润 × 数量乘数 × 兑现节奏
= 未来 2-3 年利润中枢区间
```

不同业务可以换成单台服务器、单吨材料、单 GW 产能、单条产线、单片晶圆、单柜、单客户、单项目、单产品价值量或单订单。缺数据时必须给保守/中性/乐观区间、敏感性和置信度；不得因为公司未披露分项毛利率就停止在“待验证”。

## Aggressive Repricing Spine Rule

终稿必须先建立“进攻型重定价主线”，再进入证据分层。写作时优先回答：

```text
市场正在重新定价什么？
这家公司为什么不只是旧业务修复、普通扩产或概念映射？
哪些变量会让利润斜率变陡，而不是只让收入线性增长？
最小利润模型如何从单位经济乘到利润中枢？
第一驱动参数是什么，其他变量如何服务它？
公告事实证明了哪一段，市场口径打开哪一段天花板，下一季看什么来确认？
```

高重要但未公告验证的变量，例如第二曲线客户导入、产品放量、份额提升、ASP/毛利率改善、平台复用、核心资源、产能爬坡、团队投入或少数股东权益变化，不得只写成“待验证风险”。必须写成“市场押注 -> 利润传导 -> 成立时升级 -> 不成立时降级 -> 验证指标”。如果终稿读起来像审计底稿、只剩事实边界和风险提示，说明本 skill 执行失败。

## Section Title And Judgment Rule

章节标题必须尽量给出判断，不能只写栏目名或自我缓冲。标题优先包含“变量 + 机制 + 结论方向”，让读者只看目录就能知道研究者在押什么、利润桥怎么变、跟踪点在哪里。

禁用或强烈避免以下标题句式：

```text
A 仍需验证
A 有望受益
A 不是小标签
A 具备想象空间
A 值得后续关注
合同条款变硬，但兑现闸门还没过
预期型重定价成立，但仍需等待确认
```

应改成更有交易含义的结论型标题：

```text
第二曲线若进入客户小批量，收入弹性通过产品占比和费用摊薄放大
平台复用的关键不是概念协同，而是认证、良率和交付周期能否迁移
行业蛋糕扩大只有进入 SOM 才能进利润桥，验证点是份额、订单和分业务毛利率
扩产先于订单会先消耗现金流，客户认证同步推进才会变成利润斜率
稀缺资源若兑现到台数/产能/客户份额，利润中枢从旧业务修复跃迁到新平台
单位经济和数量乘数决定重定价幅度，验证点是租金/ASP、成本、回款和实际交付
```

## Evidence Grading Timing Rule

证据边界校准放在研究末端统一执行。数据采集和主线写作阶段可以积极吸收公告、券商、行业报告、公司交流、客户/竞品线索和市场口径；终稿前再把它们分成事实底盘、经营验证、市场假设和弱线索。分级的目的不是删除变量，也不是把终稿改成审计底稿，而是决定变量在正文中的身份：事实结论、经营验证、利润弹性假设、跟踪变量或反方风险。

## Market Variable Writing Rule

对所有高重要市场变量，终稿和模块中必须采用以下写法：

```text
变量：
市场口径说什么：
证据等级：
兑现阶段：主题映射、预期型、半兑现、经营兑现，还是证伪。
如果成立：收入、毛利率、份额、客户粘性、费用率、现金流或归母利润会怎样变。
如果不成立：逻辑降级成什么，是旧业务修复、普通扩产、概念包装、成本挤压，还是主线证伪。
验证指标：公告、客户验证、订单、分业务收入/毛利率、产销量、现金流、应收存货、资本开支或管理层指引中的哪一项能验证。
终稿保留方式：
```

不合格写法：

```text
市场口径不能写成公司事实，因此不纳入正文。
新业务收入、客户和毛利率未披露，所以只能后续关注。
券商数据证据等级低，放在证据台账即可。
```

合格写法：

```text
市场正在押第二曲线客户导入和高毛利产品放量；若成立，利润桥会通过收入增量、毛利率提升和费用摊薄变陡。当前公告只验证了产能/研发/客户合作的一部分，市场口径仍需用订单、分业务毛利率和现金流验证；若长期没有经营兑现，逻辑降级为旧业务修复加概念映射。
```
