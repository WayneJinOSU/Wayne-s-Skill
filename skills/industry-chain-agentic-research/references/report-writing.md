# 连贯报告写作规范

## 写作目标

终稿要像一篇完整行业研究，而不是各 agent 摘要合集，也不是主控凭几条资料写出的行业摘要。正式交付默认包括 `final_report_full.md` 和 `executive_summary.md`；摘要不能替代完整报告。

完整报告不是“所有模块都平均展开”。完整报告应该把研究材料裁剪成一条主线：什么最重要、为什么重要、证据够不够、哪些主题只能降级或进附录。

默认读者不是行业专家。终稿必须让具备基本投资常识、但不熟悉产业链细节的普通投资人读懂逻辑。专业术语、复杂表格和公司细节可以保留，但必须先把行业生意和资金流讲顺。

读者读完应理解：

- 谁在掏钱、钱买了什么、钱最后可能留在哪些环节。
- 这个行业到底是什么生意。
- 需求和供给为什么改变价格与利润。
- 利润池留在哪个环节。
- 哪些公司能把行业变化兑现进财报。
- 当前市场定价是否已经反映预期。
- 反查模块是否找到能够推翻主线的证据。
- 什么指标会证明研究逻辑错了。

完整报告还应让读者看到产业肌理：市场规模和阶段、需求拆分、供给和价格、技术路线或商业模式、利润池迁移、竞争格局、财务映射、估值预期和反查证伪。

最重要的是让读者看到“传导过程”，而不是只看到结论。不要把传导链压缩成一行箭头；每条主线都要展开说明为什么需求会改变采购、采购如何改变产品规格、规格如何改变供需和价格、供应商如何拿份额、收入如何进财报、现金流如何验证。

## 主线优先

终稿第一屏必须给出主线：

```text
这个行业处在什么阶段，利润流向哪里，投资判断的核心变量是什么。
```

之后所有章节都服务这条主线。不要把资料按来源或 agent 分段堆放。

主线必须先来自 `report_synthesis.md`，再由主控写成 `editorial_thesis.md`。不能直接从正向研究文件跳到终稿。若 `negative_check.md` 或 `skeptic_review.md` 要求降级结论，终稿必须体现降级。

## 主笔重构

写终稿前先写 `editorial_thesis.md`。它至少回答：

```text
第一屏答案：
一句话总判断：
普通投资者逻辑桥：
本文真正要证明的 3-5 件事：
最终排序：
核心传导链：
利润桥覆盖表：
章节顺序与每章任务：
哪些内容进入正文、附录或 dossier：
反查和反方观点嵌入位置：
反拼接自检：
```

终稿结构要服从 `editorial_thesis.md`，而不是服从 agent 文件顺序。九模块和 12 类内容是覆盖要求，不是天然目录。

## 终稿落笔顺序

`final_report_full.md` 的第一屏要先回答问题，不要先铺背景：

```text
第一段：一句话总判断，包含行业阶段、利润流向和估值含义。
接下来 3-5 条：用普通语言解释谁在花钱、钱买什么、谁能留利润、哪些只是交易线/验证线，哪些不能写成强结论。
然后解释：为什么这个排序成立，最关键的证据和反证分别是什么。
```

第一屏之后必须有“普通投资者导读”或等价段落。推荐结构：

```text
先别问行业好不好，先问钱走到哪一步。
1. 谁在掏钱？
2. 钱买了什么？
3. 谁能把收入变成利润和现金流？
4. 哪些环节只是订单、项目或主题映射？
5. 当前估值提前反映了什么？
```

核心章节建议采用同一种内在节奏：

```text
判断 -> 事实证据 -> 为什么这些事实支持判断 -> 产业传导机制
-> 财务映射 -> 反证或替代解释 -> 对投资判断的含义
```

不要用“背景介绍 -> 市场规模 -> 公司列表 -> 风险提示”作为默认开头；那会天然把报告写散。

## 标题和变量写作

章节标题必须尽量写成判断句，而不是栏目名。标题要包含“变量 + 机制 + 结论方向”，让读者只看目录就知道研究者在判断什么、哪条利润链最重要、哪里会被证伪。

弱标题示例：

```text
市场规模
供需格局
竞争格局
重点公司
风险提示
```

应改成更有判断含量的标题：

```text
下游 capex 从总量扩张转向结构升级，利润更容易留在瓶颈材料和核心设备
供给释放快于需求验证的环节只能写成交易线，不能进入强主线
客户认证和价格机制决定收入能否变成毛利率，而不是只看订单口径
估值已经提前反映景气上行，后续要看现金流能否验证利润质量
```

每个高重要行业变量都必须在正文至少用一段写清楚，不能只放在表格、摘要或风险提示里。推荐段落结构：

```text
变量是什么：
证据等级是什么：
为什么市场或产业链关心它：
它如何改变需求、价格、供给、份额、毛利率、现金流或估值预期：
如果成立，判断升级到什么：
如果不成立，逻辑降级成什么：
用什么指标验证或证伪：
```

推荐写法是：“市场正在押 X；若 X 成立，产业链传导通过 A/B/C 进入价格、份额或利润池；目前证据已验证 D，仍缺 E，因此结论只能写到 F 强度。”不推荐写法是：“X 有望受益，但仍需验证。”后者只有态度，没有机制。

如果某段只是在“事实 A + 风险 B + 仍需验证 C”之间打转，没有回答“这个变量如何进入收入、毛利率、现金流、估值或排序”，必须重写。

## 内容覆盖

`final_report_full.md` 必须覆盖以下内容。行业不同可以调整顺序，也可以合并章节，但不得省略对应问题。默认采用 6-10 个一级章节或等价结构，不为了覆盖而机械拆成 12 章：

1. 核心结论：行业阶段、利润流向、投资判断变量。
2. 先讲人话：行业卖什么，谁买单。
3. 行业边界与产业链。
4. 市场规模、增速和阶段。
5. 需求拆分：增长来自哪里。
6. 产业链传导：需求如何进入采购、产品规格、订单、收入、利润和现金流。
7. 供给、产能、库存与价格。
8. 技术路线、产品结构或商业模式。
9. 利润池与竞争格局。
10. 财务映射与重点公司。
11. 估值和市场预期，作为同一章处理。
12. 反查结论：哪些看多逻辑被验证，哪些被降级。
13. 风险、证伪点和跟踪指标。

`executive_summary.md` 应压缩为 5-8 个核心判断、3-7 个证伪点和最终报告路径，不得冒充完整报告。

`plain_investor_guide.md` 应单独写给普通投资者，少用术语，不堆数据。它不是简短摘要，而是读法说明：把复杂产业链翻译成资金流、利润流、常见误区和跟踪指标。

## 表格规则

应保留的表：

- 行业规模与增速表。
- 需求拆分表。
- 供给/产能/库存表。
- 价格/成本/毛利率表。
- 产业链结构表。
- 价值链利润池表。
- 行业变量到财务指标映射表。
- 重点公司矩阵。
- 估值比较表。
- 产业链传导链表。
- 跟踪指标表。
- 证伪点表。

不应堆砌的表：

- 大量公司列表。
- 大量无解释的价格历史。
- 只展示 TAM 而不说明公司可获得份额的空间表。

每张表后必须写一句“这张表说明什么”。

关键表必须优先沉淀到 `data_tables.md`；正文可压缩展示，但不能完全省略。正文默认每个核心章节只放 0-1 张真正推动叙事的表，其他长表放附录或 `research_dossier.md`。表格后必须说明它改变了哪个判断，否则不要放进正文。

## 章节密度

完整报告每个核心章节至少包含：

```text
3-5 条事实证据或明确数据缺口
1 个结构化表格或对 data_tables.md 的引用
1 段展开的产业链传导和推理链
1 段反证/替代解释
1 句“这对投资判断意味着什么”
```

如果某章数据不足，必须写明缺少什么、当前结论如何降级、后续跟踪什么。

## 语言要求

- 第一次出现术语时用人话解释。
- 每个核心概念先给普通说法，再给专业口径。例如先写“云厂商买设备、建机房的钱”，再写 capex。
- 避免连续三段以上只出现缩写、财务指标或公司清单；中间必须穿插“这说明什么”。
- 少用“广阔空间、充分受益、确定性强、平台化”等空词。
- 结论写成条件句，不把情景假设写成事实。
- 不把行业好直接写成股票好。
- 不把短期景气直接写成长期成长。

## 利润桥写法

强结论必须写出利润桥，而不是只写“景气度提升”。最小链条是：

```text
需求/capex -> BOM/采购结构/价格 -> 供应商份额/客户验证
-> 收入确认 -> 毛利率/费用/折旧 -> 应收/存货/现金流
-> 估值隐含情景
```

如果链条中某一环没有证据，必须说明缺口，并把判断强度降级。不能把“产业链有空间”直接写成“公司利润会兑现”。

## 传导链写法

终稿必须至少完整展开 3-5 条传导链。推荐写成“故事 + 表格”的组合：

```text
普通语言起点：谁在花钱，为什么现在花。
采购变化：这笔钱进入哪些产品或服务，BOM/预算结构如何变化。
技术变化：产品规格、技术路线或商业模式为什么改变。
供需和价格：瓶颈在哪里，价格或毛利为什么能/不能守住。
公司映射：哪些公司拿份额，证据来自订单、客户、收入还是财务。
财务验证：收入、毛利率、费用、折旧、应收、存货、OCF 怎么看。
估值约束：当前价格隐含传导走到了哪一步。
断点：哪个节点最可能让判断失效。
```

每条传导链之后必须回答一句：这条链为什么能支持当前排序，为什么不能推出更强结论。

禁止写法：

```text
需求增长 -> 订单增长 -> 利润提升
```

这种写法太短，不能作为正式报告的判断依据。

## 最终排序写法

大行业终稿必须让读者知道重点在哪。推荐排序：

```text
强主线：证据和利润桥都相对完整，正文重点展开。
中强主线：方向成立但有关键数据缺口，正文展开但保留条件。
交易线：可能阶段性受益，但容易受价格、库存、订单或估值扰动。
验证线：有产业逻辑，但还缺客户、订单、财务或价格证据。
剔除线：热度高但利润桥不成立，或反证强于正证。
```

排序不是简单打分，必须说明为什么某些子环节不进入核心主线。

## 反方嵌入方式

反方观点不要只放最后。关键章节中也要穿插反证：

- 讲需求时，写需求可能不及预期的信号。
- 讲供给时，写扩产或库存的反向风险。
- 讲利润池时，写上下游挤压和替代风险。
- 讲估值时，写当前价格隐含的乐观假设。

最后再集中列出 3 到 7 个最重要证伪点。

## 反查写入方式

反查不是风险提示。终稿必须单独说明：

- 反查模块主动查了哪些反面方向。
- 找到了哪些真正反证，哪些只是一般风险。
- 哪些原本看多的结论因此被降级。
- 哪些结论在反查后仍然站得住。

如果反查没有找到硬反证，也不能写成“风险很小”，只能写“尚未找到硬反证，但仍有数据缺口”。

## 禁止行为

- 直接拼接各 agent 的小结。
- 把产业链传导压缩成箭头表或摘要句，不展开每个节点的因果关系。
- 核心判断只有“证据：xxx；结论：yyy”，没有解释为什么证据能推出结论。
- 用九模块、12 章模板或 agent 文件顺序替代主线结构。
- 跳过 `negative_check.md` 或 `report_synthesis.md` 直接写终稿。
- 跳过 `editorial_thesis.md` 直接写终稿。
- 只写行业资料，不落到财务传导。
- 只写龙头公司优点，不比较产业链环节利润。
- 只说“估值合理/偏贵”，不解释市场定价到哪一步。
- 把市场预测、券商口径或传闻写成事实。
- 把一般风险提示当成反查结论。
- 用 executive summary 冒充 full report。
- 用术语表冒充可读性；只解释词义但不解释资金流和利润流，仍视为不可读。
- 开篇直接堆排序表、公司名单或指标，不先建立普通读者能跟上的逻辑桥。
- 大行业不拆子环节，只写总论。
- 有 `data_tables.md` 却不把关键表和关键细节写进正文。
- 正文堆很多表，但没有告诉读者这些表改变了哪个判断。

## 反拼接自检

终稿完成后自查：

```text
是否先给出一条总判断，而不是先铺模块？
是否有普通投资者导读，讲清谁花钱、钱买什么、谁留利润、哪里会断？
是否有最终排序？
强结论是否都有利润桥？
每章是否在推进同一条主线？
反查是否嵌入关键章节，而不是只放最后？
表格是否服务判断，而不是为了显得厚？
被裁剪掉的材料是否进入附录或 dossier？
```

任一项不满足，回到 `editorial_thesis.md` 重写结构。
