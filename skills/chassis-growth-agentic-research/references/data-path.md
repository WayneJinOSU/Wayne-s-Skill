# 数据路径与证据优先级

## 目标

本 skill 不直接替代 `$tushare`、`$wencai-query` 或公告 PDF 处理能力。它负责确定证据链、调用顺序和口径解释。

## 证据优先级

1. 官方公告原文：年报、季报、业绩预告/快报、募投、可转债、定增、重组、关联交易、问询回复、业绩说明会。
2. `$tushare` skill：行情、市值、PE/PB、财务三表、财务指标、分红、股东、行业、历史行情。
3. `$wencai-query` skill：题材概念、市场排名、阶段涨幅、同行筛选、事件性条件、快速交叉验证。
4. 公司官网、投资者关系、交易所互动、可靠新闻和行业数据：用于补充业务进展、客户、订单、产品和产业趋势。

公告原文与结构化数据冲突时，以公告原文为准，并说明差异口径。行情和估值必须注明查询日期。

## 单公司深度研究

默认路径：

```text
1. 形成主线假设
2. 查公告索引并读取最新年报/季报/重大事项
3. 调用 `$tushare` skill 拉财务、估值、市值和历史行情
4. 调用 `$wencai-query` skill 查概念、同行和市场位置
5. 分发给业务、财务、估值、反方审查角色
6. 做利润桥和估值反推
7. 写投资人可读报告
```

## 快速判断

当用户只要快速看一只股票，可以使用本 skill 的轻量模式，也可以改用原 `$chassis-growth-research`。若使用本 skill，必须说明未启用完整 agentic 流程。

轻量模式至少覆盖：

- 旧业务底盘
- 第二成长曲线
- 当前估值是否透支

不能把轻量模式输出包装成完整深度报告。

## 横向比较或选股

当用户要求“找一批”“横向比较”“谁更强”：

```text
`$wencai-query` skill：先筛候选池和题材口径
`$tushare` skill：补财务、估值、行情和分红等结构化指标
公告原文：只抽 Top 候选或争议公司的关键公告
```

不要对所有候选公司逐一读年报，先用数据缩小范围。

## 重大事项驱动

当研究重点是重组、定增、可转债、订单、扩产、减持、实控人变化：

```text
公告原文优先
`$tushare` skill：补市值、历史行情、财务承受能力
`$wencai-query` skill：补市场反应、相关概念和同类标的
```

最终判断必须区分“事实已公告”“公司预计”“市场推演”。

## 子模块对应数据

| 模块 | 优先数据 | 辅助数据 |
| --- | --- | --- |
| 旧业务底盘 | 年报/季报、`$tushare` 财务三表、财务指标 | 行业对比、`$wencai-query` 同行筛选 |
| 第二成长曲线 | 年报业务描述、募投/扩产公告、行业数据 | `$wencai-query` 概念、同行空间对比 |
| 平台复用 | 年报产品/客户/供应链、专利/研发披露 | 公司官网、投资者关系 |
| 承接动作 | 募投、定增、可转债、订单、扩产公告 | `$tushare` 现金流/负债、`$wencai-query` 事件筛选 |
| 利润桥 | 公告分产品、`$tushare` 财务、募投效益测算 | 行业毛利率、同行净利率 |
| 远期估值 | `$tushare` 市值/估值/行情、利润桥 | `$wencai-query` 市场排名、同行估值 |

## 输出口径

每份报告至少说明：

- 使用了哪些 skill 路径。
- 最新数据日期。
- 哪些结论来自公告事实，哪些是推算。
- 哪些证据提高确信度，哪些证据降低确信度。
