# 投资人写作规范

终稿面向普通投资人，但不是低信息密度摘要。目标是先把市场正在交易的主线、天花板和利润斜率讲透，再把证据边界集中说明清楚。信息要足，逻辑要顺，事实、市场口径和情景假设要分层。

## 写作原则

- 先讲公司在产业链里做什么，再讲市场为什么重新关注它。
- 先讲市场真正交易的变量，再讲哪些已经验证、哪些只是市场口径。
- 先解释技术路线或产品代际如何改变价值量，再讲公司在哪个环节受益。
- 对产品代际、材料等级、单机价值量或 ASP 升级，必须单独解释价格和利润传导，不能只写“高端化”“结构升级”。
- 先讲订单如何变收入、收入如何变利润、利润如何变现金流，再给最终判断。
- 对天花板抬升、原材料传导、产能升级和第二曲线，正文必须有独立段落承接，不能只放在表格或反方里。
- 表格只能做索引、对比和数字承载；关键推理必须用段落讲清楚。
- `final_report.md` 不放独立估值章节，不放目标价、目标市值、PE/PEG/SOTP 表。估值输入留在 `valuation_handoff.md` 或后续估值 skill。

## 反摘要硬标准

正式 `final_report.md` 不是“把每个模块压成几段结论”的汇总稿。写作完成后必须满足：

- 正文至少 15,000 个中文字符，除非用户明确要求轻量版。
- 至少 12 个二级章节，且薄章节比例不能过高。
- 至少 8 张表或模型表，表格后必须有解释性段落。
- 每个核心章节都要包含：本章判断、关键数据或证据、机制解释、变量增强/减弱、验证指标。
- 必须显式覆盖行业周期、技术路线、产品代际、竞争客户链、原材料传导、产能承接、订单经营验证、利润桥、敏感性、跟踪日历和证伪点。
- 必须运行 `scripts/final_report_gate.py`；失败就补写，不能把失败报告当终稿交付。
- 写终稿前必须先写 `final_report_expansion_plan.md`，逐章列出对应中间文件、必写表格、必写机制、变量增强/减弱、验证指标和目标厚度。

不合格终稿的典型形态：

- 章节很多，但每章只有 2-3 段结论。
- 表格很多，但正文没有解释表格如何进入利润桥。
- 只有“已验证/待验证/风险”三类表达，没有“变量成立时利润怎么变厚，不成立时逻辑降级到哪一档”。
- 只写业务概览，没有订单到收入、收入到毛利、毛利到归母和现金流的传导链。
- 反方部分只罗列风险，没有乐观派/悲观派分歧和判决信号。

## 从中间文件到终稿的转换方法

终稿不能把各模块的结论段拼起来。每个章节都按以下顺序把中间文件“展开”成正文：

```text
1. 从对应中间文件摘出 3-6 个事实或数据。
2. 把事实放进一张小表或模型表。
3. 用 2-4 段解释这些事实之间的因果链。
4. 单独写变量增强时，收入/ASP/毛利率/份额/费用/折旧/良率/现金流/归母利润如何变化。
5. 单独写变量减弱时，逻辑降级成什么。
6. 最后写下一季或下一年的验证指标。
```

写作时优先扩“机制”，不是扩“形容词”。例如：

- 不要只写“公司受益 AI 算力高景气”；要写“AI 机柜功率和带宽提升，如何让高速连接、Busbar、液冷和电源从零件变成系统价值量”。
- 不要只写“客户认证壁垒高”；要写“认证周期、替换成本、良率、交付窗口如何影响份额和毛利率”。
- 不要只写“第二曲线打开空间”；要写“第二曲线进入报表后，哪些收入是并表，哪些是内生，哪些会被少数股东、折旧、费用和现金流扣减”。

若某章从中间文件无法抽出至少 3 个事实或数据，说明该章不应独立成章；要么回到证据收集补资料，要么并入其他章节。

## 必须写清的内容

### 1. 一句话结论

说明公司处于预期型、半兑现型、兑现型，还是证伪/高潮阶段。结论必须同时包含：

- 主线为什么成立。
- 最核心的上行变量是什么。
- 哪些经营信号会让主线升级或降级。

### 2. 公司到底做什么

用普通语言解释：

- 产品或服务是什么。
- 下游客户或应用是什么。
- 公司是材料、零部件、设备、制造、系统、渠道，还是控股/参股公司受益。
- 公司凭什么赚钱：技术、认证、规模、交付、成本、全球产能、客户关系或供应链组织。

### 3. 市场真正交易的变量

对 3 到 8 个高重要变量逐个用段落写清：

```text
变量：市场正在交易什么。
为什么会带来新增需求。
公司在哪个环节受益。
财务传导路径：量、价、毛利率、份额、费用、折旧、良率、少数股东权益或投资收益。
证据属性：公告事实、一手行业资料、市场/券商口径、情景假设或弱线索。
变量增强时：收入、ASP、毛利率、订单稳定性、现金流或利润中枢如何变化。
变量减弱时：逻辑降级成什么，是周期修复、补库涨价、低端放量、扩产先行、成本挤压，还是主线证伪。
```

不要只写“高端化带来利润弹性”“国产替代空间大”。必须说明弹性从哪里来。

核心变量段落的终点必须是判断边界，而不是证据边界。写完证据等级后继续说明：

- 如果只是普通产品放量，收入会涨但利润弹性有限。
- 如果高端产品规模量产，同样出货面积会带来更高 ASP 和毛利。
- 如果原材料涨价能转嫁，是利润放大器；如果需求降温或客户压价，就是利润挤压器。
- 如果扩产先于客户导入，会先体现为折旧、费用和爬坡成本，而不是利润。

### 4. 技术路线到价值量

如果标的处于技术迭代链条，必须解释：

```text
下游平台迭代
-> 性能瓶颈变化
-> 零部件/材料规格升级
-> 单机价值量、ASP、毛利率或用量变化
-> 供应商认证、份额、良率、产能约束
-> 订单、收入、利润和现金流兑现
```

若存在明确代际升级，必须补一段“价格与利润传导”：

```text
上一代产品价格/价值量/毛利率口径。
新一代产品价格/价值量/毛利率口径。
价差来自性能、认证、良率、稀缺、客户粘性，还是短期供需。
产品占比提升 5/10/20 个百分点，对收入和毛利的敏感性。
原材料、良率爬坡、折旧、费用和少数股东损益如何扣减。
```

若数字来自市场口径或券商模型，标成情景假设或压力测试；不要删掉，也不要伪装成公司事实。

### 5. 嵌入供应链地位

供应链地位不默认单列成“龙头/平台/认证”章节。只保留能解释利润弹性的内容，并嵌入架构天花板或订单经营验证：

- 客户认证和合作深度。
- 复杂产品量产、良率、交付周期和全球产能。
- 替代难度、替代周期和议价权。
- 客户集中和可替代风险。

如果证据只来自市场口径，要作为情景变量保留，不要写成公告事实。

供应链地位不能只写排名、份额或“龙头”。必须解释商业含义：规模、良率、交期、客户认证、多基地备份如何转化为订单稳定性、客户替换成本、产能利用率和毛利率韧性。若不能转化为这些财务变量，就降为背景信息，不抢主线。

### 6. 订单经营验证

必须写出传导链：

```text
订单 -> 排产 -> 出货 -> 收入确认 -> 利润释放 -> 现金回收
```

每一环说明证据强弱。没有数据的环节要写成可验证缺口和情景边界：若后续被验证，利润斜率怎样变陡；若长期缺失，主线降级到哪一档。

周期和承接动作也在这里完成验证：客户 capex 是否穿透到公司订单，扩产/设备/海外基地是否对应客户导入、排产、量产、良率和现金流。

### 7. 利润中枢

未来利润必须拆来源：

```text
未来利润 =
存量供应链利润
+ 新周期新增利润
- 扩张费用/折旧/财务成本/爬坡损耗
```

控股子公司要处理少数股东权益；参股公司要处理权益法收益；周期高点利润不能直接当长期中枢。

同时必须单列“天花板抬升”：

```text
历史利润高点/传统周期利润区间。
当前事实利润底盘。
新路线或第二曲线带来的利润区间。
差额来自产品代际、ASP、份额、产能利用率、良率、原材料传导、费用率摊薄还是投资收益。
```

### 8. 反方和证伪

不要到处重复“需要验证”。集中写成分歧框架和证伪逻辑：

- 乐观派在押什么变量。
- 悲观派在怀疑什么变量。
- 哪个经营信号出现，说明乐观派占优。
- 哪个经营信号出现，说明只是周期修复、补库涨价或普通产品放量。
- 哪个变量若不成立，主线会变弱或降级成周期修复、补库涨价、普通放量。
- 看什么指标。
- 指标到什么程度算证伪或降级。
- 对利润中枢和供应链身份有什么影响。

## 必须避免

- 用概念词替代解释：AI 算力、平台化、国产替代、全球交付、第二曲线。
- 用“仍需验证、后续关注、证据不足”替代判断。
- 把行业景气直接写成公司壁垒。
- 把扩产写成订单。
- 把订单写成利润。
- 把收入增长写成利润平台永久上移。
- 把季度高利润年化写成全年确定预测。
- 把券商/产业链口径写成公告事实。
- 把并表利润、权益法收益、少数股东损益重复计算。
- 把过程信息放进正文，例如主控 skill、agent 阶段、中间产物列表。

## 语言要求

- 普通投资人第一次看到术语时，要用一句话解释。
- 结论要分层：事实、推断、假设、市场口径分开。
- 强判断要写条件，不要用确定性语气遮盖证据缺口。
- 有效信息越多越好，但不能堆材料；每个数据都要服务于主线、利润桥或证伪点。
