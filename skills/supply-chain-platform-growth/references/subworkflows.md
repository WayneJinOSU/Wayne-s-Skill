# 主工作流与子工作流

本文件定义执行 SOP：研究顺序、子 skill 调度、权重排序、论证链和最终报告组装。判断框架见 [framework.md](framework.md)，数据路径见 [data-path.md](data-path.md)。具体模块检查项由各子 skill 负责，避免在主控 skill 中重复维护。

## 执行原则

正式研究报告必须覆盖 6 个核心模块：

| 模块 | 优先 skill | 作用 |
| --- | --- | --- |
| 产业周期 | `$supply-chain-cycle-capex` | 判断下游需求、客户 capex、平台迭代和周期持续性 |
| 供应链地位 | `$supply-chain-position-moat` | 判断位置、认证、交付壁垒、替代性和议价权 |
| 订单出货 | `$supply-chain-orders-shipments` | 判断订单、排产、出货、财报确认和增长质量 |
| 承接动作 | `$growth-execution-signals` | 判断扩产、融资、设备、量产、海外基地和研发投入 |
| 利润桥 | `$growth-profit-bridge` | 把订单和故事转成未来利润中枢 |
| 远期估值 | `$growth-forward-valuation` | 判断当前市值是否透支未来利润 |

按需补充：

| 场景 | 补充 skill |
| --- | --- |
| 旧业务底盘决定估值下限 | `$growth-base-business` |
| 旧能力切入新周期需要验证复用 | `$growth-platform-reuse` |
| 新赛道天花板需要展开 | `$growth-second-curve` |

每个被调用模块都要输出：

- 结论：成立、部分成立、不成立、证据不足。
- 核心证据：尽量来自公告、财报、客户 capex、订单/出货、海关、产业数据或可靠行情数据。
- 推理链：证据为什么支持结论。
- 反证或替代解释：哪些事实会削弱判断。
- 对主线影响：提高确信度、降低确信度、提高估值弹性或提高证伪风险。

轻量模式只在用户明确要求“快速判断”“简单看一下”“粗筛”时使用；必须说明未完整验证的模块和可能影响。

## 主工作流

### Step 1：提出主线假设

用一句话判断市场正在交易什么：

- 身份切换。
- 订单放量。
- 利润中枢上移。
- 估值透支。
- 错杀修复。

同时提出 3 个核心问题：

- 产业周期是否仍在上行？
- 公司是否处于关键供应链位置？
- 当前市值反映的是哪一年、哪一种利润情景？

### Step 2：框架适用性分诊

快速判断：

- 适用：产业周期、供应链地位、订单出货和利润中枢上移都有证据。
- 部分适用：周期或订单较强，但地位、利润质量或估值有缺口。
- 不适用：只是概念、普通代工、纯价格周期、金融地产消费、无利润底盘。

如果不适用，停止完整框架，输出不适用原因和更合适的框架。

### Step 3：选择证据路径

按 [data-path.md](data-path.md) 确定证据链。原则：

- 单公司深度：公告原文 + `$tushare` + `$wencai-query` + 行业/客户数据。
- 快速判断：`$tushare` + 最近公告 + `$wencai-query` + 一条关键产业证据。
- 横向比较：`$wencai-query` 先筛，`$tushare` 补财务估值，公告核验 Top 候选。
- 订单/扩产驱动：公告原文和订单/出货/海关/客户 capex 优先。

### Step 4：调用子工作流

正式研究必须覆盖 6 个核心模块。不要因为某个模块不是主矛盾就跳过；可以在最终报告里降权呈现。

### Step 5：权重排序

把模块分为：

- 主导结论：决定最终判断的 1 到 2 个模块。
- 支撑结论：提供辅助证据的 2 到 3 个模块。
- 风险约束：主要用于证伪和估值边界的模块。

权重排序不等于调用选择。

### Step 6：写出论证链

主导结论必须写成：

```text
结论：
证据：
推理：
反证/替代解释：
判断强度：
```

至少覆盖：

- 产业周期为什么能支撑订单。
- 公司为什么具备关键供应链地位。
- 订单和利润为什么能支持利润中枢上移。
- 当前估值为什么是低估、合理、偏贵或透支。

### Step 7：合成最终判断

最终判断回答：

- 公司属于预期型、半兑现型、兑现型供应链平台成长，还是不符合框架。
- 最大机会和最大风险分别来自哪里。
- 当前市值是在低估、合理定价，还是透支未来。
- 适合什么类型资金。

## 最终报告结构

1. 主结论。
2. 主线假设。
3. 框架适用性。
4. 子工作流调度表。
5. 核心论证链。
6. 利润中枢与远期估值。
7. 证伪点。
8. 最终判断。

最终报告必须区分“事实已披露”“合理推断”“情景假设”。不要把扩产等同于订单，不要把订单等同于利润，不要把季度利润年化等同于全年预测。
