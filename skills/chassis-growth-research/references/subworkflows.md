# 主工作流与子工作流

本文件只定义执行 SOP：研究顺序、子 skill 调度、检查项、输出字段和最终报告组装。判断框架、适用范围、阶段分类和常见误判放在 [framework.md](framework.md)。

## 执行原则

每次分析必须由主工作流主导。正式研究报告必须完整调用 6 个成长股子 skill，再根据证据权重合成结论；不能因为某个模块不是主矛盾就跳过。

子工作流的角色是回答主工作流提出的问题。每个被调用的子工作流都要输出：

- 结论：成立、部分成立、不成立、证据不足
- 核心证据：尽量引用财报、公告、交易所互动、公司官网、投资者关系纪要、可靠行情数据
- 推理链：证据为什么能支持结论，不能只列事实
- 反证或替代解释：哪些事实会削弱结论，是否存在更保守的解释
- 不确定性：哪些信息缺失或可能改变判断
- 对主线判断的影响：提高确信度、降低确信度、提高估值弹性、提高证伪风险

不要平均展开所有子工作流。最终报告只重点呈现最影响判断的 2 到 3 个子工作流，但调度表必须覆盖全部 6 个子 skill。若某个子 skill 数据不足，结论写“证据不足”，并列明缺口。

## 主工作流：研究调度

目标：像投研项目经理一样组织分析，而不是把七层框架切段罗列。

### Step 1：提出主线假设

先用一句话判断市场正在交易什么：

- 身份重估：公司从旧业务公司变成 AI/新能源/机器人/先进制造链条公司
- 利润中枢上移：新业务可能把未来利润抬到新的台阶
- 订单兑现：市场在验证订单、产能、客户和量产
- 估值透支：未来几年利润已经被提前定价
- 错杀修复：基本面没坏，但股价过度反映悲观预期

同时提出 3 个核心问题，例如：

- 旧业务能不能托住下限？
- 新业务是不是真能抬高天花板？
- 当前市值是否已经透支 2026-2028 年利润？

### Step 2：框架适用性分诊

快速判断：

- 适用：旧业务底盘、新成长曲线、复用能力、承接动作都基本成立
- 部分适用：有底盘或新曲线，但复用/承接/利润桥有明显缺口
- 不适用：行业逻辑不匹配，或只是概念股、周期股、金融地产、纯消费、早期创新药等

如果不适用，停止完整框架，改为说明为什么不适用。

### Step 3：选择 skill 路径

先根据任务类型选择证据链和外部 skill，再进入成长股子 skill。详见 [data-path.md](data-path.md)。

- 单公司深度研究：公告原文 + `$tushare` + `$wencai-query`。
- 快速判断：`$tushare` 为主，公告原文核验最近重大事项，`$wencai-query` 补市场位置。
- 横向比较或选股：`$wencai-query` 先筛选，`$tushare` 补结构化指标，公告只读重点公司。
- 重大事项驱动：公告原文优先，`$tushare` 补估值和财务承受能力，`$wencai-query` 补市场反应。

调度要求：

- 财务、估值、行情、市值优先调用 `$tushare` skill，并注明日期。
- 题材、概念、市场排名、同行快速筛选优先调用 `$wencai-query` skill，保留原始 `$wencai-query` 查询句子。
- 本 skill 不直接 `import tushare` 或直接调用 pywencai；只有在对应 skill 不可用、用户明确要求代码、或做兜底排障时才考虑底层包。
- 公告原文与结构化数据冲突时，以公告原文为准，并说明口径差异。

### Step 4：强制调用子 skill

正式研究报告必须调用并覆盖以下 6 个子 skill：

- `$growth-base-business`：判断旧业务下限
- `$growth-second-curve`：判断新业务上限
- `$growth-platform-reuse`：判断新业务可信度
- `$growth-execution-signals`：判断预期能否落地
- `$growth-profit-bridge`：把故事转成利润中枢
- `$growth-forward-valuation`：判断当前是否透支，并输出证伪点

轻量模式例外：

- 只有用户明确要求“快速判断”“简单看一下”“粗筛”“先不要展开”时，才允许先用 `$growth-base-business`、`$growth-second-curve`、`$growth-forward-valuation` 做最小判断。
- 轻量模式输出必须说明未完整调用的子 skill，以及这些缺口可能影响哪些结论。

### Step 5：权重排序

把子工作流分成三类：

- 主导结论：决定最终判断的 1 到 2 个子工作流
- 支撑结论：提供辅助证据的 2 到 3 个子工作流
- 风险约束：主要用于提醒证伪风险的子工作流

最终报告必须说明为什么某些子工作流权重更高。权重排序不等于调用选择；6 个子 skill 已经全部调用，只是在报告里突出主导结论的模块。

### Step 6：写出论证链

在合成最终判断前，必须先把主导结论写成 2 到 3 条论证链。每条论证链使用以下结构：

```text
结论：
证据：
推理：
反证/替代解释：
判断强度：
```

论证链至少覆盖：

- 旧业务底盘为什么成立或不成立
- 第二成长曲线为什么能抬利润中枢，而不只是抬收入或题材热度
- 当前估值为什么是低估、合理、偏贵或透支

如果某条链条只能写出证据，写不出推理或反证，说明该结论不够扎实，需要降级。

### Step 7：合成最终判断

最终判断回答：

- 公司属于预期型、半兑现型、兑现型，还是不符合框架
- 最大机会来自哪里
- 最大风险来自哪里
- 当前市值是在低估、合理定价，还是透支未来
- 适合什么类型资金

## 子工作流 A：底盘是否够硬

目标：判断公司下限。

检查项：

- 原有业务收入规模和增速
- 原有业务利润和毛利率
- 经营现金流和应收/存货压力
- 客户基础和行业地位
- 业务是否足够分散
- 没有新业务时公司是否仍能维持合理盈利

输出小结：

```text
底盘结论：
核心证据：
论证链：
主要风险：
对估值的影响：
```

## 子工作流 B：新赛道是否抬高天花板

目标：判断公司上限。

检查项：

- 新业务是什么
- 对应市场空间和未来 3-5 年增速
- 行业处于早期、中期还是成熟期
- 技术路线是否清晰
- 公司处于核心环节还是边缘环节
- 单机价值量或单位价值量是否提升

输出小结：

```text
天花板结论：
新业务：
行业空间：
公司位置：
利润中枢推理：
关键不确定性：
```

## 子工作流 C：平台复用能力

目标：判断可信度。

检查项：

- 技术是否复用
- 工艺是否复用
- 客户是否复用
- 供应链是否复用
- 产能是否复用
- 认证体系是否复用
- 是否存在完全跨界或概念化包装

输出小结：

```text
复用结论：
复用链条：
跨界风险：
替代解释：
可信度：
```

## 子工作流 D：承接动作

目标：判断预期能否落地。

检查项：

- 扩产公告
- 资本开支
- 定增、H 股、可转债、项目贷款
- 海外基地
- 设备采购
- 客户验证或定点
- 订单释放
- 量产进度
- 研发投入和人员扩张

输出小结：

```text
承接结论：
已发生动作：
尚未验证事项：
动作到业绩的传导：
落地风险：
```

## 子工作流 E：成长阶段

目标：判断市场交易的是预期差、兑现，还是透支。

分类：

- 预期型：新业务刚被发现，利润尚未明显体现
- 半兑现型：新业务已放量，市场验证持续性
- 兑现型：业绩大幅兑现，市场充分定价
- 不符合：没有底盘或没有真实新曲线

检查项：

- 新业务是否已有收入或订单
- 利润是否已经体现
- 股价是否已提前反映
- 市场仍在担心什么
- 财报是否证伪或验证市场担忧

输出小结：

```text
阶段结论：
为什么不是上一阶段：
为什么还没到下一阶段：
反证观察：
股价弹性来源：
```

## 子工作流 F：利润桥

目标：把故事转成利润中枢。

固定公式：

```text
未来利润 =
存量业务利润
+ 新业务新增利润
- 扩张期费用/折旧/财务成本/爬坡损耗
```

拆解要求：

- 存量业务利润：用历史利润、毛利率、费用率建立底盘
- 新业务收入：用 TAM、市占率、价值量、产能或订单估算
- 新业务利润率：考虑毛利率、净利率、规模效应、议价能力
- 扩张期成本：研发、折旧、人员、销售、财务、汇兑、产能爬坡

| 新业务利润 |  |  |  |  |
| 扩张成本 |  |  |  |  |

## 子工作流 G：远期估值

目标：判断当前市值是否透支。

要求：

- 使用最新股价和总市值，并注明日期
- 用 2025、2026E、2027E、2028E 利润假设计算 PE
- 明确利润假设是保守、中性还是乐观
- 说明估值能否被未来利润消化
- 说明当前市值隐含的是哪一年、哪种利润情景，避免只给 PE 表不解释含义

## 子工作流 H：证伪点

目标：找出最可能推翻逻辑的变量。

至少覆盖：

- 新业务订单或客户验证
- 毛利率和净利率
- 产能爬坡和折旧压力
- 行业技术路线变化
- 竞争格局恶化
- 当前估值是否已经透支
- 现金流、应收账款、存货等财务质量

输出要求：

按重要性列 3-5 个证伪点，并说明应该跟踪什么数据。

## 最终报告合成

最终报告不要简单拼接子工作流。需要先给出主线假设和子工作流调度表，再重点展开最影响判断的子工作流：

- 一句话结论
- 主线假设
- 子工作流调度表
- 2 到 3 条核心论证链
- 当前成长阶段
- 利润桥和估值表，如果估值是主问题
- 最关键证伪点和跟踪指标
- 对不同类型资金的适配度

建议结论语言：

```text
这家公司不是因为当期便宜而有吸引力，而是因为 X 子工作流和 Y 子工作流显示利润中枢可能上移；但 Z 子工作流显示当前估值已经计入部分预期，因此后续股价弹性取决于 A/B/C 是否继续兑现。
```
