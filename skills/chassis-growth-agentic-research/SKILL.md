# Agentic 底盘型成长股研究

## Overview

这是一个独立主控 skill，直接编排底盘型成长股研究的六个核心模块，并通过多角色研究、中间文件、反方审查和投资人写作控制输出质量。

核心判断仍然是：

```text
旧业务托底 + 新业务抬天花板 + 平台能力可复用 + 扩产融资能承接 + 未来利润能消化当前市值
```

本 skill 的目标不是生成高密度摘要，而是通过多轮独立判断和中间文件，形成普通投资人能读懂、能顺着逻辑走到结论的正式研究报告。

## When to use

适合用户要求：

- 深度分析一只 A 股制造业、AI 算力、数据中心、新能源、机器人、先进制造、半导体设备/材料等底盘型成长公司。
- 不只要核心结论，还要把业务、财务、利润桥和估值逻辑讲清楚。
- 希望使用 subagent / 多轮模型 / 多阶段研究来避免一次性摘要化输出。

不适合：

- 银行、地产、保险、纯资源周期、早期创新药、纯消费品牌。
- 用户只要“一句话看法”。如果用户要求“快速判断”“先粗筛”，可以使用轻量模式，但必须说明未完成完整 agentic 流程。

## Non-negotiables

正式研究必须同时满足以下要求：

1. 完整覆盖底盘型成长股研究的六个核心模块：
   - 旧业务底盘
   - 第二成长曲线
   - 平台复用能力
   - 承接动作
   - 利润桥
   - 远期估值与证伪点
   对应子 skill 分别是 `$chassis-growth-base-business`、`$chassis-growth-second-curve`、`$chassis-growth-platform-reuse`、`$common-growth-execution-signals`、`$common-growth-profit-bridge`、`$common-growth-forward-valuation`。能调用这些 skill 时，应优先调用；不能调用时，也必须完整覆盖同名问题。
2. 不允许直接输出终稿。必须先形成中间研究产物。
3. 每个强判断必须写清楚：

```text
证据 -> 推理 -> 反证/替代解释 -> 判断强度
```

4. 必须区分：
   - 公告或财报事实
   - 结构化数据
   - 市场口径
   - 合理推断
   - 情景假设
5. 最终报告必须面向一般投资人，解释术语、铺设推理台阶，不能只堆结论和表格。

## Agent Policy

正式研究必须触发多角色研究。业务与产业、财务与利润桥、估值与资产、反方审查这 4 个研究角色是质量控制机制，不是可选项。

在当前环境允许且 subagent 工具可用时，必须启动 subagent 分工完成这 4 个角色：

- 业务与产业 agent：负责旧业务底盘、第二成长曲线、平台复用。
- 财务与利润桥 agent：负责财务质量、经营利润、权益收益、利润桥。
- 估值与资产 agent：负责市值、PE、远期估值、股权资产折价、隐含情景。
- 反方审查 agent：专门寻找证据缺口、替代解释、估值误导和读者理解断点。

如果当前环境不允许或不支持 subagent，必须用“分阶段文件化”的方式模拟 4 个独立研究角色：每个阶段单独产出文件，下一阶段读取文件后再继续，不要只在上下文里滚动压缩。

任何角色无法获取足够信息时，必须输出“证据不足”、缺少的数据、因此需要降级的结论，不能跳过该角色。

## Workflow

### Step 1: 建立研究目录

在当前任务目录下建立中间产物目录：

```text
research_artifacts/<股票代码或公司简称>/
  <标的简称或代码>_question.md
  <标的简称或代码>_evidence_index.md
  <标的简称或代码>_business_industry.md
  <标的简称或代码>_financial_profit_bridge.md
  <标的简称或代码>_valuation_asset.md
  <标的简称或代码>_skeptic_review.md
  <标的简称或代码>_report_outline.md
  <标的简称或代码>_final_report.md
```

### Step 2: 主线假设与适用性分诊

先写入 `<标的简称或代码>_question.md`：

- 市场正在交易什么：身份重估、利润中枢上移、订单兑现、估值透支、错杀修复。
- 需要验证的 3 个核心问题。
- 公司是否适用底盘型成长股框架；如不适用，停止完整流程并说明更合适框架。

### Step 3: 证据收集

写入 `<标的简称或代码>_evidence_index.md`，按可获得性覆盖：

- 公告原文：年报、季报、募投、定增/可转债、重大合同、关联交易、问询回复。
- `$tushare`：行情、市值、估值、财务三表、财务指标、历史行情。
- `$wencai-query`：市场概念、同行排名、题材口径、阶段涨幅、市场关注点。
- 公司官网、交易所互动、投资者关系或可靠行业资料。

证据优先级和数据路径见 [references/data-path.md](references/data-path.md)。

### Step 4: 多角色研究

正式研究中，主控必须覆盖六个底盘型成长股核心模块，但可以由不同 agent 或阶段分工完成。

主控必须在 `<标的简称或代码>_report_outline.md` 或终稿前保留一张覆盖表：

| 核心模块 | 对应子 skill | 负责角色/阶段 | 结论强度 | 证据缺口 |
| --- | --- | --- | --- | --- |
| 旧业务底盘 | `$chassis-growth-base-business` | 业务/财务 |  |  |
| 第二成长曲线 | `$chassis-growth-second-curve` | 业务 |  |  |
| 平台复用能力 | `$chassis-growth-platform-reuse` | 业务 |  |  |
| 承接动作 | `$common-growth-execution-signals` | 业务/财务 |  |  |
| 利润桥 | `$common-growth-profit-bridge` | 财务 |  |  |
| 远期估值与证伪点 | `$common-growth-forward-valuation` | 估值/反方 |  |  |

业务与产业研究写入 `<标的简称或代码>_business_industry.md`：

- 旧业务底盘是否够硬。
- 第二成长曲线是否抬高天花板。
- 平台复用是否证明新业务可信。
- 公司处于预期型、半兑现型、兑现型，还是不符合框架。

财务与利润桥研究写入 `<标的简称或代码>_financial_profit_bridge.md`：

- 收入、毛利率、费用率、净利率、经营现金流、应收、存货。
- 存量业务利润、新业务新增利润、扩张成本。
- 如果存在重要投资收益或股权资产，必须拆开经营利润与投资收益。
- 保守、中性、乐观三种情景。

估值与资产研究写入 `<标的简称或代码>_valuation_asset.md`：

- 最新市值、PE/PB、历史估值、同行估值。
- 当前市值反推 2025、2026E、2027E、2028E PE。
- 当前市值隐含哪一年、哪种利润情景。
- 若公司持有重要上市/非上市资产，必须说明资产价值、折价原因、可变现性和估值波动。

反方审查写入 `<标的简称或代码>_skeptic_review.md`：

- 哪些结论证据不足。
- 哪些地方把事实直接跳成了结论。
- 哪些估值口径可能误导普通投资人。
- 哪些术语、链条或假设会让一般投资人读不懂。
- 最关键 3 到 5 个证伪点。

### Step 5: 写作提纲

写入 `<标的简称或代码>_report_outline.md`，不要直接写终稿。提纲必须回答：

- 读者第一眼需要先理解什么。
- 哪几个概念必须用人话解释。
- 哪些结论需要先铺业务逻辑，再给财务数字。
- 哪些表格应该保留，哪些应该转成解释性段落。
- 最终报告的 6 到 10 个一级章节。

### Step 6: 面向普通投资人的终稿

读取全部中间文件后，再写 `<标的简称或代码>_final_report.md`。

终稿结构优先使用下列顺序；若内容重叠，可以合并章节，但不能省略对应逻辑：

1. 一句话结论：公司是预期型、半兑现型、兑现型，还是已经充分定价。
2. 先讲人话：这家公司到底靠什么赚钱。
3. 为什么市场关注它：新周期、新业务、身份变化。
4. 旧业务底盘：下限在哪里。
5. 第二成长曲线：上限来自哪里。
6. 平台复用与承接动作：为什么不是单纯讲故事。
7. 财务质量与利润桥：当期利润如何走到远期利润。
8. 估值与资产折价：当前贵不贵、市场在定价什么。
9. 反方观点和证伪点。
10. 最终判断：机会、风险、适合什么类型资金。

## Output Discipline

最终回复用户时，除非用户要求只看终稿，否则要说明：

- 新生成了哪些中间文件。
- 哪些 agent / 阶段完成了哪些任务。
- 六个底盘型成长股核心模块如何被覆盖，最好给出覆盖表。
- 终稿文件路径。
