# Agentic 编排细则

## 总控职责

总控不是亲自写完所有分析，而是像投研项目经理一样组织研究。

总控必须完成：

- 提出主线假设。
- 判断框架适用性。
- 建立中间产物目录。
- 调度数据工具和研究角色。
- 处理不同角色之间的冲突。
- 决定最终报告里哪些模块主导结论，哪些只是支撑或风险约束。

## 推荐文件结构

```text
research_artifacts/<股票代码或公司简称>/
  <标的简称或代码>_question.md
  <标的简称或代码>_evidence_index.md
  <标的简称或代码>_business_industry.md
  <标的简称或代码>_financial_profit_bridge.md
  <标的简称或代码>_valuation_asset.md
  <标的简称或代码>_skeptic_review.md
  <标的简称或代码>_report_outline.md
  <标的简称或代码>_final_report.md
```

同一轮研究必须使用同一个文件名前缀，格式为 `<标的简称或代码>_`。例如：`中孚实业_evidence_index.md` 或 `600595_evidence_index.md`。

## 角色分工与原子 Skill 映射

本 skill 是总控增强版，不废弃原有子 skill。正式研究中，优先让对应角色调用或吸收这些原子 skill 的问题清单：

| 原框架模块 | 原子 skill | 主要负责角色 | 作用 |
| --- | --- | --- | --- |
| 旧业务底盘 | `$growth-base-business` | 业务与产业 agent、财务与利润桥 agent | 判断下限、现金流、客户和财务质量 |
| 第二成长曲线 | `$growth-second-curve` | 业务与产业 agent | 判断上限、产业空间、价值量和身份重估 |
| 平台复用能力 | `$growth-platform-reuse` | 业务与产业 agent | 判断新业务可信度，防止硬跨界 |
| 承接动作 | `$growth-execution-signals` | 业务与产业 agent、财务与利润桥 agent | 判断扩产、融资、订单、量产是否支撑预期 |
| 利润桥 | `$growth-profit-bridge` | 财务与利润桥 agent | 把故事转成未来利润中枢 |
| 远期估值与证伪 | `$growth-forward-valuation` | 估值与资产 agent、反方审查 agent | 判断当前市值是否透支，并输出证伪点 |

主控可以把多个原子 skill 合并给同一个 agent，但不能省略任何模块。若某个模块证据不足，应写“证据不足”，而不是从报告中消失。

正式研究必须触发 4 个研究角色：业务与产业、财务与利润桥、估值与资产、反方审查。当前环境允许且工具可用时，必须用真实 subagent；当前环境不允许或不支持时，必须用阶段文件模拟这些角色。区别只在执行载体，不在研究步骤是否存在。

### 业务与产业 agent

任务边界：

- 旧业务底盘
- 第二成长曲线
- 平台复用能力
- 成长阶段

不能做：

- 不要给最终投资结论。
- 不要用估值结论反推业务合理性。

输出文件：`<标的简称或代码>_business_industry.md`

输出结构：

```text
业务结论：
旧业务底盘：
第二成长曲线：
平台复用链条：
成长阶段：
证据 -> 推理 -> 反证 -> 判断：
普通投资人容易误解的地方：
对主线判断的影响：
```

### 财务与利润桥 agent

任务边界：

- 收入、毛利率、费用率、净利率。
- 经营现金流、应收、存货、合同负债、资本开支。
- 存量利润、新业务利润、扩张成本。
- 经营利润和投资收益拆分。

输出文件：`<标的简称或代码>_financial_profit_bridge.md`

输出结构：

```text
财务结论：
收入和利润质量：
现金流和资产负债表风险：
经营利润与非经营/投资收益拆分：
利润桥：
保守/中性/乐观情景：
最敏感变量：
反证或失效条件：
对主线判断的影响：
```

### 估值与资产 agent

任务边界：

- 最新股价、市值、PE/PB、历史估值。
- 当前市值反推未来 PE。
- 同行估值和市场排名。
- 重要股权资产、资产折价、母公司折价、流动性折价。

输出文件：`<标的简称或代码>_valuation_asset.md`

输出结构：

```text
估值结论：
当前市值和估值口径：
远期 PE 反推：
当前市值隐含情景：
资产价值与折价：
为什么不能直接等同于无风险套利：
关键证伪点：
对主线判断的影响：
```

### 反方审查 agent

任务边界：

- 只挑错，不负责把报告写漂亮。
- 检查证据链跳步、估值误导、数据口径混用、风险遗漏。
- 检查普通投资人是否能理解。

输出文件：`<标的简称或代码>_skeptic_review.md`

输出结构：

```text
最大漏洞：
证据不足的结论：
可能误导的估值口径：
替代解释：
读者理解断点：
必须补充或降级的判断：
最关键 3-5 个证伪点：
```

## 无 Subagent 时的模拟方式

如果当前环境不能启动 subagent，不要退回一次性写作。必须按以下方式模拟：

1. 先写 `<标的简称或代码>_business_industry.md`，只处理业务。
2. 再读取该文件，写 `<标的简称或代码>_financial_profit_bridge.md`，只处理财务。
3. 再读取前两个文件，写 `<标的简称或代码>_valuation_asset.md`，只处理估值。
4. 再读取前三个文件，写 `<标的简称或代码>_skeptic_review.md`，只挑错。
5. 最后读取所有文件，写提纲和终稿。

关键是每一轮都有独立文件，而不是把所有思考留在上下文里。

如果某一阶段信息不足，仍然要生成对应文件，并写明：

- 已查到的证据。
- 缺少的数据。
- 哪些判断只能降级为“证据不足”或“正在验证”。
- 下一步需要补什么信息。

## 冲突处理

当不同角色给出冲突判断时，总控必须显式处理：

- 业务 agent 认为天花板高，但财务 agent 找不到利润兑现路径：降级为“有空间但利润桥待验证”。
- 财务 agent 认为利润增长快，但估值 agent 认为已充分定价：结论不能写“便宜”，应写“兑现要求高”。
- 估值 agent 发现资产价值倒挂，但反方 agent 提醒流动性或母公司折价：必须解释折价来源，不能写成无风险套利。
- 公告和市场口径冲突：公告优先，市场口径作为交易情绪或同类比较。

## 最终整合

终稿不是拼接四份中间稿。总控必须重新排序：

- 先解释公司怎么赚钱。
- 再解释市场为什么关注。
- 再解释这个故事如何落到财务。
- 最后讨论估值和证伪。

结论语言要体现条件性：

```text
这家公司不是因为当期便宜而有吸引力，而是因为旧业务底盘和新业务承接动作让未来利润中枢存在上移可能；但当前市值已经计入一部分乐观预期，后续弹性取决于利润桥能否继续兑现。
```
