# 正式行业深度报告交付合同

## 目的

本文件约束所有行业、赛道和产业链研究，不针对任何单一行业。它解决的问题是：agentic 流程可能很完整，但终稿被压缩成投研摘要，或被写成多个 agent 小结的拼接稿，缺少行业研究报告的厚度和主线。

正式研究必须输出三类成品：

```text
<prefix>_final_report_full.md      # 完整行业深度报告，不能过薄
<prefix>_plain_investor_guide.md   # 普通投资者导读，解释报告逻辑，不替代 full report
<prefix>_executive_summary.md      # 给投资人的摘要，不替代 full report
<prefix>_transmission_map.md       # 产业链传导图谱，解释需求如何进入利润和估值
<prefix>_final_report_expansion_plan.md # 终稿扩写施工图和反摘要补写记录
```

如需兼容旧流程，可额外输出 `<prefix>_final_report.md`，但它不能替代 `final_report_full.md`。
如需保留大量资料、长表、公司清单或被终稿裁剪掉的模块材料，可额外输出 `<prefix>_research_dossier.md`。dossier 是资料卷，不是正式终稿。

## 可读性合同

正式报告默认面向“具备基本投资常识、但不熟悉本产业链细节的普通投资人 + 专业投资人”。因此必须同时满足两层阅读：

1. **普通读者能顺着逻辑读懂。** 在 `final_report_full.md` 第一屏或紧随第一屏加入“普通投资者导读”或等价段落，先用人话解释：谁掏钱、钱买什么、谁能留利润、哪些只是订单/主题、估值已经反映什么。
2. **专业读者能复核证据。** 正文保留关键数据、表格、证据链、利润桥和反查；长表进入 `data_tables.md` 或附录。

`<prefix>_plain_investor_guide.md` 必须单独生成，默认包含：

```text
一句话结论：
资金流图：谁花钱 -> 买什么 -> 谁确认收入 -> 谁留利润 -> 估值验证
3-5 个普通投资者最容易误解的点：
最终排序的普通语言解释：
后续跟踪指标：
如何阅读完整报告：
```

禁止把术语表当作可读性。真正的可读性是让读者理解“为什么这个证据推出这个排序”。

## 传导逻辑合同

正式研究必须生成 `<prefix>_transmission_map.md`，并把其中的关键链条写入 `final_report_full.md` 正文。传导逻辑不能只是一张表或一行箭头，必须展开成文字说明。

每条核心主线至少按以下链条展开：

```text
下游需求/capex 为什么发生
-> 客户采购决策和 BOM/预算结构如何变化
-> 产品规格、技术路线或商业模式如何变化
-> 哪个环节出现供给瓶颈或议价权
-> 哪些供应商能拿份额，证据是什么
-> 订单如何交付并确认收入
-> 收入如何变成毛利、费用、折旧和利润
-> 应收、存货、合同负债、OCF/FCF 如何验证质量
-> 当前估值隐含传导链走到了哪一步
-> 哪个节点最容易断
```

大行业至少要完整展开：

- 1 条强主线传导链。
- 1 条中强主线传导链。
- 1 条交易线传导链，说明为什么不能写强。
- 1 条验证线传导链，说明缺哪些证据。

如果终稿只写“需求强、供给紧、利润留在某环节”，没有展开上述节点，只能标记为 preliminary draft。

## 判断依据展开

每个核心判断必须采用可读的证据链，而不是只列结论：

```text
判断：
事实证据：
为什么这些事实能支持判断：
产业机制：
传导到公司财报的路径：
为什么不能推出更强结论：
反证/替代解释：
判断强度：
```

如果只有事实没有“为什么”，读者无法复核；如果只有机制没有事实，就是叙事。两者都不能作为正式报告的核心判断。

## 大行业分解

如果研究对象是大行业、复合赛道或宽产业链，必须先写 `<prefix>_subsegment_map.md`，拆成 3-7 个子环节。拆分维度由行业决定，可以按上中下游、产品形态、客户类型、技术路线、商业模式或利润池位置。

每个子环节必须回答：

```text
卖什么：
谁买单：
需求来自哪里：
供给瓶颈在哪里：
价格怎么定：
利润留在哪：
哪些公司能兑现：
关键反证：
```

不能只在总表里列子环节，必须在正文中至少对核心子环节展开。

## 数据表沉淀

正式研究必须生成 `<prefix>_data_tables.md`。数据不足也要建表，并写清缺口。通用表包括：

| 表 | 用途 |
| --- | --- |
| 行业规模与增速表 | 说明行业处于什么阶段，空间如何形成收入，而不是只写 TAM |
| 需求拆分表 | 按客户、场景、区域、渗透率、替换、capex、出口或库存周期拆需求 |
| 供给/产能/库存表 | 说明扩产、瓶颈、库存、开工率、交付周期和退出难度 |
| 价格/成本/毛利率表 | 说明价格机制、成本曲线、招投标/长协/现货/批价和毛利率变化 |
| 产业链利润池表 | 比较各环节毛利、净利、ROIC、现金流、议价权和风险 |
| 重点公司矩阵 | 分兑现型、半兑现型、预期型、概念型，列证据和缺口 |
| 估值比较表 | 口径、历史分位、同行、隐含利润情景和安全边际 |
| 利润桥覆盖表 | 从需求/capex 走到份额、收入确认、毛利率、现金流和估值，用于判断强结论能否成立 |
| 产业链传导链表 | 把下游需求、采购、技术路线、瓶颈、供应商份额、财务确认和估值连接起来 |
| 跟踪指标表 | 高频/低频指标、领先财报的信号和证伪阈值 |

## 章节草稿

每个正向研究角色除研究备忘录外，必须提供可入正文的章节草稿。执行方式二选一：

1. 在角色文件末尾增加 `可入正文的章节草稿：`。
2. 额外写入 `chapter_drafts/<prefix>_<chapter>.md`。

章节草稿不是角色小结，必须有正文叙述、关键表格、证据引用和反证/替代解释。

## 正式报告内容覆盖

`final_report_full.md` 必须覆盖以下 12 类内容。12 章结构只是兜底模板，不是默认目录；正式终稿默认采用 6-10 个一级章节或等价结构，章节顺序应由 `<prefix>_editorial_thesis.md` 决定：

1. 核心结论：行业阶段、利润流向、投资判断变量。
2. 行业定义与边界：卖什么、谁买单、排除哪些相似但不同的生意。
3. 市场规模、增速和阶段：历史、当前、远期，不把 TAM 直接写成利润。
4. 需求拆分：按客户/场景/区域/渗透率/capex/出口/库存周期拆开。
5. 供给、产能、库存和价格：利润会不会被扩产、库存或价格下行吃掉。
6. 技术路线、产品结构或商业模式：行业靠什么迭代或收费。
7. 产业链利润池：利润留在哪，为什么能守住，是否迁移。
8. 竞争格局和重点公司：龙头、追赶者、边缘玩家，谁能兑现。
9. 财务映射：行业变量如何进入收入、毛利率、费用、现金流和资产负债表。
10. 估值与市场预期：当前价格隐含到哪一年、哪种利润情景。
11. 反查反证：哪些正向逻辑被保留、降级或推翻。
12. 跟踪指标和证伪点：3-7 个关键证伪点，含领先指标。

## 主笔重构

`report_synthesis.md` 之后必须生成 `<prefix>_editorial_thesis.md`。它是正式终稿的写作指令，必须回答：

```text
第一屏答案：
一句话总判断：
普通投资者逻辑桥：
本文真正要证明的 3-5 件事：
最终排序：
核心传导链：
利润桥覆盖表：
正文结构与每章任务：
正文 vs 附录/dossier 分配：
反查与反方观点嵌入位置：
反拼接自检：
```

没有 `editorial_thesis.md`，或该文件只是重复 `report_synthesis.md`，不得进入 `final_report_full.md`。

## 终稿复杂度档位

正式报告的长度和章节密度由行业复杂度决定，不由模型默认压缩决定。写 `report_outline.md` 时必须先判断研究对象属于哪一档，并在 `final_report_expansion_plan.md` 复核：

| 档位 | 适用条件 | 正文密度 | 闸门 profile |
| --- | --- | --- | --- |
| `compact` | 窄赛道、子环节不超过 3 个、强主线集中、证据有限或用户明确要求简洁 | 8,000-10,000 中文字，8-10 个二级章节，至少 6 张关键表 | `--profile compact` |
| `standard` | 多数行业研究默认档；需要完整覆盖需求、供给、价格、利润池、公司映射、估值预期和反证 | 10,000-14,000 中文字，10-12 个二级章节，至少 8 张关键表 | 默认或 `--profile standard` |
| `complex` | 大行业、复合产业链、子环节不少于 5 个、利润池迁移复杂、公司兑现路径多 | 12,000-18,000 中文字，12-16 个二级章节，至少 10 张关键表 | `--profile complex` |
| `long-form` | 用户要求长篇深度、券商深度版、对标 25-40 页报告，或研究对象有多技术路线/多地区/多商业模式 | 18,000-30,000 中文字，章节和表格按证据复杂度增加 | `--profile long-form` |

不得为了满足字数重复铺陈；闸门失败时优先补机制、证据、传导链、财务映射、反证和跟踪指标。若内容确实简单，应在 `report_outline.md` 和 `final_report_expansion_plan.md` 说明为什么使用 `compact`，并运行对应 profile。

## 终稿扩写蓝图

写 `final_report_full.md` 前必须先写 `<prefix>_final_report_expansion_plan.md`。它不是读者交付物，而是防止摘要化的施工图，至少包含：

```text
复杂度档位与理由：
子环节数量、核心变量数量、利润池数量、公司映射数量：
终稿目标篇幅和章节密度：
每章绑定的中间文件：
每章必须展开的事实证据：
每章必须解释的产业机制：
每章必须保留或引用的表格：
每章必须展开的传导链节点：
每章必须写入的反证/替代解释：
每章如何映射到收入、毛利率、费用、现金流、估值预期或证伪指标：
正文 vs 附录/dossier 分配：
可能变成摘要的薄弱章节：
闸门补写记录：
```

每个核心章节至少绑定 1 个中间文件，不能凭上下文记忆写。凡 `data_tables.md`、`transmission_map.md` 或 `report_synthesis.md` 里的核心变量，如果没有进入正文，必须在扩写蓝图说明删除、合并或降级原因。

## 利润桥与最终排序

强判断必须走通利润桥：

```text
需求/capex -> BOM/采购结构/价格 -> 供应商份额/客户验证
-> 收入确认 -> 毛利率/费用/折旧 -> 应收/存货/现金流
-> 估值隐含情景
```

大行业必须做最终排序。推荐分层如下，可按行业替换名称，但必须保留“谁是主线、谁被降级、谁只进观察池”的功能：

```text
强主线：
中强主线：
交易线：
验证线：
剔除线：
```

不能进入利润桥的子环节、公司或主题，不得写成强结论；可以写入附录、观察池或证伪指标。

## 密度要求

正式报告不能只做结论归纳。每个核心章节至少包含：

```text
3-5 条事实证据或明确数据缺口
1 个结构化表格或对已有表格的引用
1 段推理链
1 段反证/替代解释
1 句“这对投资判断意味着什么”
```

如果某章因数据不足无法满足，必须写：

```text
证据不足：
缺少的数据：
当前只能做出的降级判断：
后续应跟踪的数据：
```

正文默认每个核心章节只放 0-1 张真正推动叙事的表。长表、公司清单、口径解释和被裁剪的模块材料进入 `data_tables.md`、附录或 `research_dossier.md`；正文中的每张表后必须写明它改变了哪个投资判断。

## 行业阶段判断

正式报告必须判断行业阶段，并解释证据。可选阶段包括：

- 导入期
- 渗透加速期
- capex 上行期
- 供给释放期
- 价格下行期
- 出清期
- 成熟期
- 多阶段并存

阶段判断必须连接到利润和估值：阶段不只是行业描述，而是解释利润率、现金流和估值口径的入口。

## Report Synthesis 增强

`report_synthesis.md` 除冲突、证据强弱和降级结论外，必须新增：

```text
最终排序：
利润桥覆盖表：
普通投资者逻辑桥与术语翻译：
终稿必须展开的 10-20 个细节：
终稿必须完整展开的 3-5 条产业链传导链：
必须保留的表格：
每章必须引用的证据：
不得压缩掉的产业机制：
正文 vs 附录分配：
证据卡到正文位置映射：
可能导致终稿离散的拼接风险：
executive_summary 写作指令：
plain_investor_guide 写作指令：
final_report_full 写作指令：
```

如果 synthesis 没有这些字段，主控不能直接写 full report。

## 防过薄验收

以下任一情况出现，输出只能标记为 `executive_summary` 或 `preliminary draft`，不能称为正式行业深度报告：

- 没有 `final_report_full.md`。
- 没有 `<prefix>_final_report_expansion_plan.md`。
- 没有 `<prefix>_plain_investor_guide.md`。
- 没有 `<prefix>_transmission_map.md`。
- 没有 `data_tables.md`。
- 大行业没有 `subsegment_map.md`。
- 正文只有结论，没有展开需求、供给、价格、利润池和财务映射。
- 正文没有完整展开产业链传导链，只用箭头、排序表或摘要句替代机制说明。
- 核心判断只有结论和事实，没有解释“为什么这些事实支持该判断、为什么不能推出更强结论”。
- 只列重点公司，没有说明行业变量如何进入收入、利润、现金流和估值。
- 反查只在最后列风险，没有嵌入正文章节。
- 估值只说便宜/贵，没有反推隐含情景。
- 终稿明显短于中间产物，且未说明哪些内容进入附录。
- 没有 `editorial_thesis.md`，或 `editorial_thesis.md` 没有最终排序和利润桥覆盖。
- 第一屏没有直接回答“这个行业现在该怎么看、重点看哪、哪些不能写成强结论”。
- `final_report_full.md` 没有普通读者逻辑桥，开篇直接堆专业术语、长表或公司名单。
- 终稿结构明显复制模块顺序、agent 顺序或 12 章模板，且没有主线递进理由。
- 正文堆了大量表格和公司清单，但没有说明它们改变了哪个投资判断。
- 没有运行 `scripts/final_report_gate.py`，或闸门失败后没有补写并复跑。

正式深度报告模式下，`final_report_full.md` 生成后必须运行反摘要闸门：

```bash
python3 /Users/a/.codex/skills/industry-chain-agentic-research/scripts/final_report_gate.py \
  "/absolute/path/<prefix>_final_report_full.md"
```

若 `report_outline.md` 或 `final_report_expansion_plan.md` 已明确判定为 `compact`、`complex` 或 `long-form`，运行对应 profile：

```bash
python3 /Users/a/.codex/skills/industry-chain-agentic-research/scripts/final_report_gate.py \
  "/absolute/path/<prefix>_final_report_full.md" --profile complex
```

默认 `standard` 闸门检查中文字符数、二级章节数、表格数、薄章节比例、深章节数量、行业研究核心概念覆盖和每章财务/验证信号。闸门只防止摘要化，不鼓励堆字数；失败时把失败项写入 `final_report_expansion_plan.md` 的“闸门补写记录”，补写机制和证据后复跑。

## 附录优先

正文要有判断，但不能因为怕堆资料而丢掉证据。正式研究可以把大表放入：

```text
<prefix>_appendix_data.md
<prefix>_company_matrix.md
<prefix>_source_notes.md
<prefix>_research_dossier.md
```

附录不替代正文，正文必须保留关键证据和结论。
