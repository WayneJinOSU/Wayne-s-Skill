# 投资人写作规范

本文件只负责 `final_report` 的写法。终稿结构、复杂度、gate 和章节硬约束见 [final-report.md](final-report.md)；QA 和出版清理见 [qa-gates.md](qa-gates.md)。`final_report` 是唯一正式报告，不另写券商版副本。

## 写作目标

把中间研究材料改写成投资人能直接判断的因果链：

```text
最大预期差
-> 第一驱动参数
-> 单位经济或最小经营单元
-> 数量乘数和兑现节奏
-> 利润中枢区间
-> 验证、降级和证伪
```

所有章节都服务同一条第一主线。旧业务、第二曲线、行业蛋糕、平台复用、竞争客户链、承接动作、利润桥和反方审查，只能承担四种功能：增强主线、校准主线、证伪主线或提供经营下限。

## 开篇：首页投资判断

终稿开篇先扩写 `investment_logic_card` 的“投资人首页判断”，不要从行业背景或风险提示起笔。开篇 800-1200 字必须包含：

- “我们认为”或等价判断句。
- 最大预期差和第一驱动参数。
- 公司公告、财报、交易所文件、客户/竞品资料、券商假设或产业链线索中的事实锚。
- 利润传导：变量如何影响收入、毛利率、费用率、折旧、财务费用、现金流或归母利润。
- 验证和降级：未来 2-4 个季度看什么；若变量弱化，逻辑降级到旧业务修复、普通扩产、主题映射还是主线证伪。

推荐句式：

```text
我们认为，公司的核心分歧不在于 A，而在于 B 这个第一驱动参数能否通过 C 传导到利润斜率。公司公告/财报/产业链线索已经验证了 D，但 E 仍需用订单、分业务毛利率、现金流或客户认证进度确认。若 B 成立，利润桥主要通过 F 变陡；若 B 弱化，逻辑应降级到 G。
```

## 底盘型叙事链

底盘型成长股默认按以下顺序组织论证：

```text
旧业务给下限
-> 新变量改变利润斜率
-> 公司用平台、产能或客户资源放大变量
-> 扩产、认证、产品结构决定兑现节奏
-> 折旧、财务费用和现金流决定利润穿透率
-> 竞争和证据边界决定上限与降级路径
```

不要机械套用供应链平台公司的“下游需求 -> 技术路线 -> 价值量 -> 客户认证 -> 订单放量 -> 利润兑现”顺序，除非标的确实主要由订单、客户认证和单件价值量驱动。对底盘型材料或制造公司，先讲旧业务下限和单位经济/利润斜率，再讲客户、需求和产能如何放大或约束这条主线。

## 段落语法

核心段落用以下顺序写：

```text
判断
-> 事实锚
-> 利润参数
-> 变量增强时的变化
-> 变量弱化时的降级
-> 验证指标
```

每个核心段落至少落到一个利润参数：单位经济、数量乘数、兑现节奏、份额、ASP、毛利率、费用率、折旧/财务成本、税率、少数股东或现金回收。不能落到参数的内容，压缩成背景、放入跟踪项或删除。

表格只能承载数字、对比和索引。删除表格后，正文仍应能讲清本章判断；如果不能，先补段落再保留表格。

## 市场变量写法

高重要市场变量不能因为不是公告事实就删掉。应降级保留为市场口径、券商假设、产业链线索或待验证变量，并写清：

```text
市场正在交易什么
证据属性是什么
若成立，收入/毛利率/费用率/现金流/归母利润如何变化
若不成立，逻辑降级到哪一档
用什么指标继续验证
```

不要用“尚未披露、后续关注、无法确认”结束核心变量段落。改成变量增强/减弱时结论如何变化。

## 自然来源表达

内部索引只用于定位，不进入读者可见正文。正式报告应把内部材料转换成自然来源：

| 内部材料 | 正文表达 |
| --- | --- |
| `Fact-ID/Lead-ID` | 公司公告/财报披露、交易所文件显示、客户或竞品资料显示 |
| 市场线索 | 产业链线索显示、券商假设的核心分歧在于、市场正在交易的变量是 |
| 模块结论 | 我们认为、核心分歧在于、利润弹性来自、验证点是 |
| 证据不足 | 当前更适合作为情景变量处理；进入更高情景需要观察 X |

`final_report` 正文不得出现 `Fact-ID`、`Lead-ID`、`skill`、`subagent`、`gate`、`handoff`、`final_report`、`brokerage_report`、`扩写蓝图`、`闸门`、`终稿`、`研究阶段候选`、`正确写法`、`普通投资人最容易误判`、`本报告只做` 等内部痕迹。

## 研报语言边界

研报化意味着结论前置、变量聚焦、证据分层和风险后置校准，不意味着写正式估值结论。`final_report` 不单列估值章节，不写目标价、目标市值、买卖评级、正式 PE/PEG/SOTP/DCF 结论，也不讨论“当前贵不贵”。估值因子、情景准入、三表/FCF/UFCF 驱动和模型消费规则放到两轮 gate 后 handoff，由后续估值 skill 承接。

语言要清楚但不装腔。少用“显著、充分、确定性、长期受益、空间广阔、壁垒深厚”这类强词，除非证据确实支撑。术语第一次出现时用一句话解释，并立即落到本公司的利润模型上。

## 写作前自检

写终稿前确认四件事：

- 开篇能否脱离全文单独作为投资人首页阅读。
- 每章第一段是否说明本章影响哪个核心利润参数。
- 高重要市场变量是否进入正文主线、利润桥和跟踪体系，而不是只留在证据台账或反方审查。
- 反方审查是否后置为降级路径和证伪指标，而不是把第一章改成保守摘要。
