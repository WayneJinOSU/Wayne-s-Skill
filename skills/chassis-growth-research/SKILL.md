# 底盘型成长股研究

## Overview

用于分析“传统业务有现金流和利润底盘，同时因 AI、数据中心、新能源、机器人、先进制造等新周期出现第二成长曲线”的制造业公司。

核心问题：

```text
这家公司是否具备旧业务托底、新业务抬天花板、平台能力可复用、扩产融资能承接、未来利润能消化当前估值的特征？
```

## When to use

适合研究 A 股制造业二次成长公司，例如电力电子、PCB、高速连接、线缆、光通信、散热/液冷、新能源车零部件、工控自动化、机器人零部件、半导体设备/材料、先进封装材料、AI 服务器零部件。

不适合直接用于银行、地产、保险、纯消费品牌、纯资源周期、早期创新药、纯互联网软件、没有旧业务底盘的概念股。

## Workflow

把这个 skill 当作主控 skill 使用。它负责提出主线假设、判断适用性、选择子 skill、处理证据冲突并形成最终判断；具体深挖交给独立子 skill。不要机械地调用全部子 skill，只调用主线问题真正需要的模块。

1. 主工作流第一步：形成研究假设。
   - 用一句话提出市场正在交易什么，例如身份重估、利润中枢上移、订单兑现、估值透支或错杀修复。
   - 同时写出需要验证的 3 个核心问题。
2. 主工作流第二步：框架适用性分诊。
   - 快速判断公司属于适用、部分适用、不适用。
   - 如果不适用，停止完整框架，只输出不适用原因和更合适的框架。
3. 主工作流第三步：选择 skill 路径。
   - 优先确定需要公告原文、`$tushare`、`$wencai-query`，还是三者组合。
   - 财务、估值、市值、历史行情优先调用 `$tushare` skill，不在本 skill 中直接调用底层 tushare Python 包。
   - 题材、概念、市场排名、同行快速筛选优先调用 `$wencai-query` skill。
   - 年报/季报、募投、重组、关联交易、交易所问询和业绩说明会以公告原文为最高证据。
4. 主工作流第四步：选择子工作流。
   - `$growth-base-business`：旧业务底盘、现金流、客户、行业地位和下限。
   - `$growth-second-curve`：新赛道天花板、产业趋势、价值量和上限。
   - `$growth-platform-reuse`：技术、客户、工艺、产能、供应链和认证体系复用。
   - `$growth-execution-signals`：扩产、融资、设备、客户验证、订单、量产和研发投入。
   - `$growth-profit-bridge`：存量利润 + 新业务利润 - 扩张成本的情景测算。
   - `$growth-forward-valuation`：当前市值反推远期 PE、估值透支和证伪点。
5. 主工作流第五步：权重排序。
   - 判断最终报告里最重要的 2 到 3 个子工作流，不要平均用力。
   - 明确哪些证据提高确信度，哪些证据降低确信度。
6. 主工作流第六步：合成结论。
   - 回答公司是在讲故事、正在验证、正在兑现，还是已经充分定价。
   - 最终结论必须围绕“未来利润中枢是否上移、当前估值是否已透支”展开。

## Output format

输出时优先按下面结构：

1. 主结论：一句话说明预期型、半兑现型、兑现型，或不符合框架。
2. 主线假设：市场正在交易什么，以及这次研究最需要验证的 3 个问题。
3. 子工作流调度表：列出调用了哪些子工作流、为什么调用、结论权重。
4. 关键证据：按重要性写 2 到 3 个最核心证据，不要平均铺开。
5. 利润桥与估值：只有在数据足够或估值是主问题时展开情景测算。
6. 证伪点：最关键的 3 到 5 个，并说明跟踪指标。
7. 最终判断：机会、风险、适合什么类型资金、当前是错杀还是透支。

## Data discipline

分析具体公司时，优先使用最新公告、年报/季报、交易所互动、公司官网、投资者关系纪要和可靠市场数据。涉及当前市值、股价、PE、业绩预告、订单、扩产、融资等可变信息时，需要核对日期并说明口径。

skill 路径和证据优先级见 [references/data-path.md](references/data-path.md)。当任务涉及 A 股结构化行情、财务、估值或市场语义/同行筛选时，按该路径调度 `$tushare` skill 与 `$wencai-query` skill。
