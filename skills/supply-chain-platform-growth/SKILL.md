# 供应链平台型成长股研究

## Overview

用于分析“不是最高技术定义者，但在大产业周期里承担关键制造、交付、材料、设备、连接、散热、电源或配套角色”的平台型供应链公司。

核心问题：

```text
这家公司是否从普通制造/代工/零部件身份，切换为大产业周期里的关键供应链交付平台？
订单和出货是否能推动利润中枢上移？
当前市值是否已经透支未来 2 到 3 年利润？
```

一句话方法：

```text
先判断产业周期，再判断供应链地位，再判断利润中枢是否上移，最后判断估值是否透支。
```

## Relationship To Other Growth Skills

把本 skill 当作独立主控 skill 使用。它和 `chassis-growth-research` 并列：

- `chassis-growth-research`：侧重旧业务底盘、第二成长曲线、平台复用、利润桥和远期估值，适合“旧业务托底 + 新业务二次成长”的制造业公司。
- `supply-chain-platform-growth`：侧重身份切换、产业周期、供应链地位、订单出货领先指标和利润年份切换，适合“产业周期爆发 + 关键交付平台”的公司。

正式研究时优先复用已有子 skill：

- `$supply-chain-cycle-capex`：验证产业周期、客户资本开支、产品平台迭代和周期持续性。
- `$supply-chain-position-moat`：验证供应链位置、客户认证、交付壁垒、可替代性和议价权。
- `$supply-chain-orders-shipments`：验证订单、排产、出货、收入确认、营运质量和现金流。
- `$growth-execution-signals`：验证扩产、客户、订单、量产、海外基地和研发投入。
- `$growth-profit-bridge`：建立旧平台利润、新周期新增利润、扩张成本和情景测算。
- `$growth-forward-valuation`：反推当前市值隐含的 2025-2028 年利润情景和估值透支。
- `$growth-base-business`：当公司旧业务底盘对估值下限很重要时调用。
- `$growth-platform-reuse`：当公司从旧产品切入新周期需要验证能力复用时调用。
- `$growth-second-curve`：当新周期空间、价值量和天花板需要展开时调用。

## Framework Fit

适合：

- AI 服务器、GPU/ASIC 服务器、云厂商算力硬件交付平台。
- 高速 PCB、HDI、IC 载板、服务器主板、交换机相关供应链。
- 液冷散热、电源、连接器、高速线缆、光模块、光器件。
- 先进封装设备/材料、半导体设备/材料里的关键配套环节。
- 消费电子、通信设备、工业制造公司切入 AI 硬件、机器人、新能源等大周期。

不适合：

- 银行、地产、保险、纯资源周期、纯消费品牌。
- 纯软件互联网、早期创新药、生物科技。
- 无收入利润底盘、只有概念没有订单和产能承接的题材股。
- 主要靠价格周期而不是交付放量和平台复用的公司。

如果公司只是普通代工，且没有客户认证、交付规模、复杂产品量产、供应链组织或全球产能优势，必须降级为一般制造公司，不要强行套用本框架。

## Workflow

### 1. Form The Main Hypothesis

先用一句话写清楚市场正在交易什么：

```text
市场正在交易：身份切换 / 订单放量 / 利润中枢上移 / 估值透支 / 错杀修复。
```

同时提出 3 个必须验证的问题：

1. 产业周期是否足够大且仍在上行？
2. 公司是否处在关键供应链位置，而不是可替代普通供应商？
3. 当前股价反映的是哪一年、哪一种利润情景？

### 2. Judge Industry Cycle First

先看下游大周期，不要先看 PE。

正式研究优先调用 `$supply-chain-cycle-capex`，并把它的结论作为产业周期部分的主证据。

重点回答：

- 下游需求是否真实爆发，而不是公司单方面讲故事。
- 客户资本开支是否扩张，周期是否仍有 2 到 3 年可见度。
- 产品平台是否迭代，价值量、复杂度或单位价格是否提升。
- 是否存在第二增长线，例如服务器之外的高速交换机、网络设备、液冷、电源或海外基地。

对 AI 算力链，优先跟踪：

- 全球云厂商资本开支。
- GPU/ASIC 服务器、AI 服务器、交换机、网络设备出货。
- 下游客户排产、供应链验证和平台迭代节奏。
- 产业链价格、交期、库存和竞争扩产。

### 3. Identify Market Identity Shift

判断市场是否正在把公司从旧身份切到新身份。

常见身份切换：

```text
消费电子/通信设备/普通制造
-> AI 服务器/高速网络/算力硬件供应链平台
```

必须区分三种状态：

| 状态 | 特征 | 估值含义 |
| --- | --- | --- |
| 未切换 | 市场仍按旧业务和普通制造定价 | 估值弹性有限 |
| 正在切换 | 订单、利润或客户信号推动身份重估 | 弹性最大 |
| 已充分切换 | 市场已按新身份定价，需要利润继续验证 | 风险转向杀估值 |

不要把“公司属于 AI 概念”直接等同于“身份切换成功”。身份切换必须由订单、出货、利润率、客户结构或市场定价行为共同验证。

### 4. Judge Supply Chain Position

判断公司在产业链中是哪种角色：

正式研究优先调用 `$supply-chain-position-moat`，并把它的结论作为供应链地位部分的主证据。

- 核心技术定义者。
- 关键零部件/材料供应商。
- 平台型制造与交付商。
- 普通代工或可替代供应商。

供应链平台型公司的壁垒通常不是芯片 IP 或品牌心智，而是：

- 大客户认证。
- 大规模交付能力。
- 复杂产品量产经验。
- 全球产能和供应链组织。
- 良率、交期、成本控制。
- 与客户平台迭代同步的工程能力。

结论要保持两边约束：

```text
不能因为没有最高技术定价权就否定它。
也不能把关键供应链平台拔高成顶级科技垄断公司。
```

### 5. Track Orders And Shipments Before Financials

这类公司财报滞后，领先信号通常来自订单和出货。

正式研究优先调用 `$supply-chain-orders-shipments`，并把它的结论作为订单出货和增长质量部分的主证据。

优先建立传导链：

```text
订单领先 -> 排产/出货 -> 收入确认 -> 利润释放 -> 财报确认 -> 机构上修 -> 股价反应
```

关键观察维度：

| 观察维度 | 作用 |
| --- | --- |
| 订单、排产、出货 | 判断需求是否真实放量 |
| 海关或产业链数据 | 验证订单强度和出口节奏 |
| 客户资本开支 | 判断需求持续性 |
| 产品平台迭代 | 判断 ASP、复杂度和价值量 |
| 产能扩张、海外基地 | 判断能否承接订单 |
| 存货、应收、合同负债 | 判断增长质量和回款 |
| 毛利率、净利率 | 判断是否只增收不增利 |
| 经营现金流 | 判断是否靠垫资换增长 |

把订单信号分为：

- 强信号：明确订单、量产、客户项目、出货放量、财报利润验证。
- 中信号：扩产、设备采购、认证进展、客户合作表述。
- 弱信号：概念、传闻、没有数量和金额的笼统表述。

### 6. Test Profit Center Uplift

核心不是某一年利润高低，而是正常利润平台是否换台阶。

用季度利润年化时必须写明它只是情景假设：

```text
Q4 x 4 不是精确预测，而是检验公司是否进入新利润平台的压力测试。
```

必须回答：

1. 高利润季度是偶发，还是新平台起点？
2. 下一年订单能否支撑全年高利润？
3. 后年是否还能继续增长，而不是下一年见顶？
4. 收入增长是否带来毛利率、净利率或规模效应改善？
5. 现金流、应收、存货是否支持利润质量？

正式报告必须建立利润桥：

```text
未来利润 =
旧平台利润
+ 新周期新增利润
- 扩张费用/折旧/财务成本/爬坡损耗
```

### 7. Classify Growth Stage

按预期差和兑现程度分类：

| 类型 | 特征 | 股价弹性 | 主要风险 |
| --- | --- | ---: | --- |
| 预期型供应链成长 | 身份刚切换，订单和利润尚未充分体现 | 最大 | 证伪风险高 |
| 半兑现型供应链成长 | 订单和利润部分体现，市场继续上修未来 | 中高 | 业绩波动杀估值 |
| 兑现型供应链成长 | 利润大幅兑现，市场已经充分乐观 | 下降 | 增速放缓和估值压缩 |

判断时不要只说公司好坏，要说明它处在哪个阶段，以及预期差是否还在。

### 8. Price By Profit Year

估值时必须明确当前股价反映的是哪一年利润。

常见运动：

```text
当年利润定价
-> 下一年利润定价
-> 后年利润定价
-> 增速放缓后回到当年利润定价
-> 杀估值
```

正式研究必须输出：

- 当前市值对应 2025、2026E、2027E、2028E PE。
- 每一年利润假设属于保守、中性还是乐观。
- 当前股价隐含的是哪一年、哪一种利润情景。
- 如果只有乐观情景才能消化估值，必须标记为透支或偏贵。

### 9. Define Failure Signals

重点跟踪会让“供应链平台重估”退潮的信号：

- 下游资本开支放缓。
- AI 服务器或核心产品订单增速下降。
- 客户平台切换导致订单空窗。
- 毛利率或净利率下滑。
- 应收账款和存货大幅增加。
- 经营现金流持续弱于利润。
- 客户压价、竞争对手扩产、份额流失。
- 产能爬坡低于预期，海外基地进度不及预期。
- 市场重新按普通代工/制造估值。

证伪点要和跟踪指标绑定，不要只写抽象风险。

## Output Format

优先按下面结构输出：

1. 主结论：公司属于预期型、半兑现型、兑现型供应链平台成长，还是不符合框架。
2. 主线假设：市场正在交易什么，以及最需要验证的 3 个问题。
3. 框架适用性：说明为什么适用、部分适用或不适用。
4. 产业周期：下游需求、客户资本开支、产品迭代和周期持续性。
5. 身份切换：旧身份、新身份、切换证据和市场是否已经充分定价。
6. 供应链地位：产业链角色、壁垒来源、可替代性和议价能力。
7. 订单出货：领先指标、财报验证、收入利润传导和数据缺口。
8. 利润中枢：季度年化检验、利润桥、保守/中性/乐观情景。
9. 远期估值：当前市值反映到哪一年利润，是否透支。
10. 证伪点：3 到 5 个最关键变量和跟踪指标。
11. 最终判断：机会、风险、适合资金类型、当前是错杀、合理还是透支。

## Data Discipline

分析具体公司时，优先使用最新公告、年报、季报、业绩预告、投资者关系纪要、交易所互动、公司官网、客户资本开支数据和可靠市场数据。

涉及股价、市值、PE、订单、出货、扩产、业绩预告、客户资本开支等可变信息时，必须核对日期并说明口径。

强判断必须区分：

- 事实已披露。
- 合理推断。
- 情景假设。

不要把扩产等同于订单，不要把订单等同于利润，不要把季度利润年化等同于全年预测，也不要把 AI 概念等同于估值切换成功。
