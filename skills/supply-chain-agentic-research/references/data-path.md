# 数据路径与证据优先级

## 目标

本 skill 不直接替代 `$tushare`、`$wencai-query`、公告 PDF 处理、客户 capex、券商/产业链线索或海关/出货数据能力。它负责确定证据链、调用顺序、市场变量扩表和口径解释。

## 证据优先级

1. 官方公告原文：年报、季报、业绩预告/快报、订单、合同、募投、定增、可转债、问询回复、业绩说明会。
2. 客户和行业一手资料：客户财报、资本开支指引、电话会、供应链公开数据、产品平台路线、行业协会或权威机构数据。
3. `$tushare` skill：股价、市值、PE/PB、财务三表、财务指标、现金流、历史行情、行业和股本。
4. `$wencai-query` skill：题材概念、市场排名、同行筛选、阶段涨幅、事件性条件和快速交叉验证。
5. 公司官网、投资者关系、交易所互动、可靠新闻和券商纪要：用于补充客户、订单、产品、产能和产业趋势。
6. 海关、出货、排产和产业链跟踪数据：用于验证订单是否真实放量。若来源不是官方，必须说明可靠性和口径限制。

公告原文与结构化数据冲突时，以公告原文为准，并说明差异口径。行情和估值必须注明查询日期。

## 市场变量分层

供应链成长股常先由产业链和券商口径交易，后由公告和财报验证。公告确定事实底座，市场口径识别弹性变量；关键变量要进入主线、天花板和利润斜率模型，同时标明证据等级。

证据台账必须把重要变量分为四类。表格只用于索引和检查，不能替代正文推理：

| 类型 | 例子 | 写法 |
| --- | --- | --- |
| 已证实事实 | 公告订单、财报利润、公司明确披露的产品和产能 | 可进入主结论 |
| 一手行业/客户资料 | 客户 capex、终端路线图、行业协会数据、公司官网产品参数 | 可支撑产业周期和技术路线 |
| 市场/券商口径 | 券商模型假设、专家问答、产业链调研、交易热词 | 作为情景变量进入主线和敏感性 |
| 弱线索/传闻 | 无法交叉验证的份额、良率、客户认证、订单排产 | 作为跟踪变量，说明若成立/不成立的影响 |

对高重要市场变量，必须用段落写清：

- 市场正在交易什么变量。
- 为什么会带来新增需求：下游变化如何改变用量、价值量、规格、认证门槛或供需关系。
- 公司在哪个环节受益：公司是材料、零部件、设备、制造、系统、渠道，还是控股/参股公司受益。
- 财务传导路径：影响量、价、毛利率、份额、费用、折旧、良率、少数股东权益、投资收益中的哪几项。
- 证据属性：公告事实、客户/行业一手资料、市场/券商口径、情景假设或弱线索。
- 变量增强时：收入、ASP、毛利率、订单稳定性、现金流或利润中枢会怎样变化。
- 变量减弱时：逻辑会降级成什么，是周期修复、补库涨价、低端放量、扩产先行、成本挤压，还是主线证伪。

## 单公司深度研究

默认路径：

```text
1. 形成主线假设
2. 读最新年报/季报/业绩预告/重大事项公告
3. 调用 `$tushare` skill 拉财务、现金流、历史行情、市值和 PE/PB；估值数据只用于 handoff，不进入终稿正式结论
4. 调用 `$wencai-query` skill 查概念、同行、阶段涨幅、市场排名和交易热词
5. 抓取券商研报、产业链调研、专家纪要、客户/竞品资料、官网参数和原材料价格，建立市场变量索引
6. 先写 `<标的>_market_variables_map.md`，用段落写清核心变量的影响逻辑
7. 若存在技术路线或产品代际快速变化，读取 `technology-route-value-chain.md`，并写 `<标的>_architecture_value_ceiling.md`
8. 对关键原料和产能升级，分别写 `<标的>_raw_material_price_chain.md` 和 `<标的>_capacity_second_curve.md`
9. 分发给订单经营验证、利润桥、估值接力输入和终审 QA；周期、地位和承接动作默认嵌入主攻模块
10. 做利润桥和估值接力输入；后续正式估值可交给 `$growth-stock-valuation`
11. 写投资人可读报告，并在终稿前检查关键市场变量是否被无意删掉
```

## 快速判断

当用户只要快速看一只股票，可以使用本 skill 的轻量模式，并必须说明未启用完整 agentic 流程。

轻量模式至少覆盖：

- 市场真正交易的 3 到 5 个变量。
- 技术路线或价值量变化。
- 订单出货与经营验证。
- 利润中枢和最敏感变量。

不能把轻量模式输出包装成完整深度报告。

## 横向比较或选股

当用户要求“找一批”“横向比较”“谁更强”：

```text
`$wencai-query` skill：先筛候选池、题材口径和市场排名
`$tushare` skill：补财务、估值、现金流和行情指标
公告原文：只抽 Top 候选或争议公司的关键公告
产业数据：按环节比较客户 capex、价值量、订单和竞争扩产
```

不要对所有候选公司逐一读年报，先用数据缩小范围。

## 订单/扩产/业绩驱动

当研究重点是订单、扩产、业绩预告、海外基地、客户验证：

```text
公告原文优先
订单/排产/出货/海关数据辅助
`$tushare` skill 补市值、行情、现金流和财务承受能力
`$wencai-query` skill 补市场反应和同类标的
```

必须区分“事实已公告”“公司预计”“产业链推断”“市场情绪”。

## 子模块对应数据

| 模块 | 优先数据 | 辅助数据 |
| --- | --- | --- |
| 市场变量扩表 | 券商研报、产业链调研、交易热词、官网参数、竞品文件 | wencai 市场表现、专家纪要、可靠媒体 |
| 架构/技术路线与天花板 | 客户路线图、产品规格、终端平台、竞品资料 | 券商模型、专家纪要、公司官网参数 |
| 原材料价格链 | 铜、树脂、玻纤、化工品、金属、芯片等原料价格和供需 | 同行调价、库存、存货跌价、客户价格条款 |
| 产能升级与第二曲线 | 基地产能、募投、在建工程、海外基地、控股/参股公司 | 客户导入、设备采购、人员研发、产能利用率 |
| 订单出货与经营验证 | 订单公告、量产、出货、海关、排产、客户认证、份额、替代周期 | 财报收入、毛利率、存货、应收、合同负债、现金流 |
| 周期/地位/承接嵌入检查 | 客户 capex、客户财报、行业出货、客户认证、份额、替代周期、扩产、设备、海外基地、研发投入 | 默认回填到市场变量、架构/天花板、产能升级和订单经营验证，不单列模块 |
| 利润桥 | 财报分产品、毛利率、订单、产能和成本 | 行业毛利率、同行净利率、募投效益 |
| 估值接力输入 | `$tushare` 市值/估值/行情、利润桥 | 一致预期、同行估值、历史估值、市场阶段；正式估值用 `$growth-stock-valuation` |

## 关键数据口径

必须注明：

- 股价、市值和估值的查询日期。
- 财报或业绩预告的报告期和披露日。
- 客户 capex 的来源、币种、会计年度和是否为公司相关业务。
- 订单金额是否含税、是否框架协议、是否已量产或收入确认。
- 海关/出货数据的统计口径、商品编码、地区和时间范围。
- 利润假设是事实、合理推断还是情景假设。

## 证据强度分级

| 证据等级 | 例子 | 用法 |
| --- | --- | --- |
| 强证据 | 公告订单、财报利润、客户 capex 指引、明确量产出货 | 可支撑主结论 |
| 中证据 | 扩产、设备、认证、客户合作、行业出货趋势 | 可支撑推断，并需要订单、财报或现金流验证 |
| 弱证据 | 传闻、概念、未量化表述、笼统进入供应链 | 只能作为线索 |

对“高重要、低证据”的变量，不能直接删除；必须作为情景变量保留，并在正文说明成立时的利润弹性、不成立时的降级路径，在终审 QA 中说明证伪方式。

## 输出口径

每份报告至少说明：

- 使用了哪些 skill 路径：公告、`$tushare`、`$wencai-query`、客户 capex、海关/出货、网页核验。
- 最新数据日期和报告期。
- 哪些结论来自公告事实，哪些来自推算或情景假设。
- 哪些来自券商、产业链或市场口径，为什么保留、如何验证。
- 哪些证据提高确信度，哪些证据降低确信度。

## 兜底原则

如果客户 capex、订单或海关数据缺失，不要直接写成强兑现。应把结论改成情景判断：若后续出现订单、排产、毛利率和现金流验证，主线升级；若长期缺失，逻辑降级为周期修复、补库涨价、普通放量或主题映射，并列出需要跟踪的数据。
