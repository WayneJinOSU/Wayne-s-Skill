# 内置模块问题清单

本文件承接已删除的供应链平台子模块和成长股通用子模块框架。主 skill 不再依赖独立子 skill，但正式研究必须按这些清单执行；如果数据不足，写“证据不足”和缺口，而不是省略模块。

## 产业周期与客户 Capex

目的：回答公司所处的大产业周期是否真实、足够大、仍在上行，并且能支撑未来 1 到 3 年订单和利润中枢。

优先读取：

- 下游客户资本开支、预算指引、财报和电话会。
- 行业出货量、市场规模、增速、价格、库存和交期。
- AI 服务器、交换机、PCB、液冷、电源、连接器、光模块、先进封装等产品平台迭代信息。
- 公司公告、投资者关系纪要、产能规划和客户结构。
- 可变数据必须核对日期和口径。

工作流：

1. 确认公司绑定的下游周期：AI 算力、数据中心、服务器、交换机、机器人、新能源、先进封装或其他。
2. 判断需求是真爆发、温和增长、存量替换，还是阶段性拉货。
3. 拆客户 capex：主要客户是谁，资本开支是否扩张，扩张用于什么产品或区域。
4. 判断产品平台迭代：价值量、复杂度、单机用量、ASP、良率要求是否提升。
5. 判断周期阶段：早期放量、中期扩散、后期拥挤、见顶回落。
6. 检查供给侧：竞争对手扩产、产能过剩、价格下降、库存累积。
7. 给出周期可见度：6-12 个月、1-2 年、2-3 年，或证据不足。
8. 列出会让周期判断失效的反证条件。

关键问题：

- 下游需求来自真实资本开支，还是题材预期？
- 客户 capex 是总量扩张，还是结构迁移？
- 公司产品价值量是否随新平台提升？
- 本轮周期是刚开始、正在兑现，还是已经被充分预期？
- 需求持续性靠新增算力、替换升级、海外扩张，还是一次性拉货？
- 供给扩张是否会压缩价格和利润率？

输出字段：

```text
周期结论：
绑定的下游周期：
客户 capex 证据：
产品平台迭代：
需求持续性：
周期阶段：
供给和价格风险：
反证或替代解释：
数据缺口：
对主线判断的影响：
```

纪律：不要把“行业空间大”直接等同于“公司订单会增长”。必须说明从客户 capex 到公司产品需求的传导链；不要只引用远期 TAM，供应链平台股更需要判断未来 4 到 8 个季度的订单可见度和周期位置。

## 供应链地位与壁垒

目的：回答公司在产业链中是否处于关键位置，壁垒来自哪里，是否具备阶段性不可替代性和合理议价能力。

优先读取：

- 公司客户结构、认证资质、合作年限、重点客户项目。
- 产品复杂度、良率、交付周期、产能规模、全球基地。
- 同行业竞争格局、份额、扩产、替代供应商。
- 毛利率、净利率、价格趋势和客户集中度。
- 公司公告、年报、投资者关系纪要、官网和可靠产业数据。

工作流：

1. 定义产业链角色：核心技术定义者、关键零部件/材料、平台型制造交付、普通代工或边缘配套。
2. 判断客户重要性：是否进入头部客户核心供应链，是否有多客户扩散。
3. 拆壁垒来源：认证、规模、交付、良率、成本、产能、供应链组织、全球布局、工程响应。
4. 判断可替代性：替代难度、替代周期、竞争对手能力、客户双供/多供策略。
5. 判断议价权：公司能否分享价值量提升，还是只承担放量但被压价。
6. 检查客户集中风险：单一客户或单一平台变化是否会显著影响收入利润。
7. 给出地位等级：核心平台、关键供应商、重要配套、普通供应商、证据不足。
8. 列出会削弱地位判断的反证。

地位等级：

| 等级 | 特征 | 估值含义 |
| --- | --- | --- |
| 核心平台 | 多客户、多产品、复杂交付、难替代 | 可获得身份重估 |
| 关键供应商 | 单一环节强壁垒，客户认证较深 | 有重估弹性但受环节限制 |
| 重要配套 | 有订单和产能，但替代者较多 | 主要看利润兑现 |
| 普通供应商 | 产品同质、客户压价、替代容易 | 不宜给平台溢价 |

输出字段：

```text
地位结论：
产业链角色：
客户和认证：
壁垒来源：
可替代性：
议价权：
客户集中风险：
反证或替代解释：
数据缺口：
对主线判断的影响：
```

纪律：不能因为公司不是最高技术定义者就直接否定它；供应链平台公司的壁垒可以来自客户认证、规模交付、良率和全球组织能力。也不能把“大客户供应商”自动等同于“平台型公司”，必须验证替代难度、利润率和多产品/多客户扩散。

## 订单出货与经营质量

目的：回答订单和出货是否真实放量，能否传导到收入、利润和现金流，并提前验证财报尚未完全体现的增长。

优先读取：

- 公司公告、订单、合同、中标、客户项目、量产和出货表述。
- 排产、产能利用率、海关、产业链跟踪、渠道或第三方数据。
- 收入、毛利率、净利率、存货、应收、合同负债、经营现金流。
- 产能扩张、设备采购、在建工程、海外基地、人员和研发投入。
- 可变信息必须核对日期、周期和口径。

工作流：

1. 汇总订单和出货证据，并按强、中、弱信号分级。
2. 建立传导链：订单 -> 排产 -> 出货 -> 收入确认 -> 利润释放 -> 现金回收。
3. 判断订单是新增长、补库存、集中交付，还是一次性大单。
4. 检查收入质量：是否伴随毛利率、净利率改善，还是只增收不增利。
5. 检查营运质量：应收、存货、合同负债、经营现金流是否匹配增长。
6. 检查产能承接：现有产能、扩产进度、良率和海外基地是否足够。
7. 判断财报滞后程度：订单是否已在收入利润中体现，还是仍有确认空间。
8. 列出订单逻辑失效的反证条件。

信号分级：

| 信号等级 | 例子 | 判断方式 |
| --- | --- | --- |
| 强信号 | 明确订单金额/数量、量产、出货放量、财报利润验证 | 可作为主线证据 |
| 中信号 | 扩产、设备、客户认证、合作框架、产能爬坡 | 只能说明承接，需后续验证 |
| 弱信号 | 概念、传闻、笼统进入供应链、未量化表述 | 不能单独支撑结论 |

质量检查：

- 收入增长是否快于或慢于订单表述。
- 毛利率是否因产品升级、规模效应或良率改善而提升。
- 存货增长是否合理，是否存在积压风险。
- 应收账款是否异常扩张，回款是否拖累现金流。
- 经营现金流是否长期弱于利润。
- 合同负债或预收是否支持订单可见度。
- 产能扩张是否可能带来折旧和爬坡损耗。

输出字段：

```text
订单出货结论：
强/中/弱信号分级：
订单到收入利润传导：
财报确认程度：
毛利率和净利率质量：
存货、应收和现金流：
产能承接情况：
反证或替代解释：
数据缺口：
对主线判断的影响：
```

纪律：不要把“扩产”当作订单，不要把“订单”直接当作利润，不要把“收入增长”直接当作高质量增长。供应链平台股的关键是订单放量必须最终体现为利润中枢上移，并且不能靠明显恶化的应收、存货和现金流换增长。

## 承接动作

目的：回答公司是否用真金白银和经营动作承接新周期预期。

优先读取：扩产公告、募投项目、定增/H 股/可转债、资本开支、在建工程、设备采购、海外基地、客户验证、订单、量产进度、研发投入和人员变化。

工作流：

1. 汇总已经发生的承接动作。
2. 区分资本开支、融资、客户验证、订单、量产和研发投入。
3. 判断动作是实质承接还是口径表述。
4. 检查动作和新周期收入/利润是否匹配。
5. 说明动作如何传导到收入和利润，避免把融资、扩产本身当作兑现。
6. 列出尚未验证的反证点。
7. 给出落地等级：强承接、初步承接、弱承接、无承接。


## 利润桥

目的：回答订单、出货和身份切换能不能转化为未来利润中枢，未来利润能否消化当前估值。

优先读取：历史利润、业务分部毛利率、费用率、产能、订单、价格、出货量、资本开支、折旧、研发费用、财务费用和管理层指引。

固定公式：

```text
未来利润 =
旧平台利润
+ 新周期新增利润
- 扩张费用/折旧/财务成本/爬坡损耗
```

工作流：

1. 建立旧平台利润底盘。
2. 估算新周期收入：客户 capex、价值量、份额、产能、订单或出货。
3. 估算新周期利润率：毛利率、净利率、规模效应、议价能力。
4. 估算扩张成本：研发、折旧、人员、销售、财务、汇兑和爬坡损耗。
5. 判断高利润季度是偶发，还是新平台起点；不要把季度利润年化当精确预测。
6. 说明从当期利润到远期利润的跃迁路径和最敏感变量。
7. 列出会让利润桥失效的反证条件。
8. 做保守、中性、乐观三种情景。

| 扩张成本 |  |  |  |  |

输出字段：

## 远期估值

目的：回答当前市值到底是在低估未来、合理定价，还是已经透支 2 到 3 年利润。

优先读取：最新股价、总市值、股本、2025 利润、2026E-2028E 利润假设、同行估值、历史估值和关键证伪指标。

工作流：

1. 确认当前股价和总市值，并注明日期和来源。
2. 用当前市值反推 2025、2026E、2027E、2028E PE。
3. 判断利润假设属于保守、中性还是乐观。
4. 判断估值状态：低估、合理、偏贵、透支。
5. 说明当前市值隐含的是哪一年、哪种利润情景，避免只给 PE 表不解释含义。
6. 列出最关键 3 到 5 个证伪点和跟踪指标。

## 身份切换与阶段判定

总控在整合时必须判断市场身份是否切换，以及公司处于哪一阶段。

常见身份路径：

```text
消费电子/通信设备/普通制造
-> AI 服务器/高速网络/算力硬件供应链平台
```

身份切换状态：

| 状态 | 特征 | 估值含义 |
| --- | --- | --- |
| 未切换 | 市场仍按旧业务和普通制造定价 | 估值弹性有限 |
| 正在切换 | 订单、利润或客户信号推动身份重估 | 弹性最大 |
| 已充分切换 | 市场已按新身份定价，需要利润继续验证 | 风险转向杀估值 |

成长阶段：

| 类型 | 特征 | 股价弹性 | 主要风险 |
| --- | --- | ---: | --- |
| 预期型供应链成长 | 身份刚切换，订单和利润尚未充分体现 | 最大 | 证伪风险高 |
| 半兑现型供应链成长 | 订单和利润部分体现，市场继续上修未来 | 中高 | 业绩波动杀估值 |
| 兑现型供应链成长 | 利润大幅兑现，市场已经充分乐观 | 下降 | 增速放缓和估值压缩 |
| 不符合框架 | 无真实订单、无平台地位或无利润底盘 | 不适用 | 题材退潮 |

身份切换必须由订单、出货、利润率、客户结构或市场定价行为共同验证；不要把“AI 概念”直接等同于“身份切换成功”。
