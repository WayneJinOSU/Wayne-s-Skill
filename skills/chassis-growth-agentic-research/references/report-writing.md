# 投资人写作规范

终稿要让一般投资人理解“为什么这个结论成立”，而不是只看到研究员式摘要。写作目标是把市场正在交易的主线、旧业务下限、第二曲线天花板、平台复用、竞争/客户验证、承接动作、利润斜率和证伪路径讲透；正式估值留给 `$growth-stock-valuation`。

## 写作原则

- 先解释公司靠什么赚钱，再讲市场为什么重新定价。
- 先讲市场真正交易的变量，再讲哪些已经验证、哪些只是市场口径。
- 先讲业务机制，再给财务数字和利润斜率。
- 先把上行模型写完整，再用证据等级约束；不要一上来就把报告写成审计底稿。
- 对客户份额、订单、ASP、毛利率、单机价值量、平台复用等市场口径，不要删除；要降级为影响因子、情景变量或跟踪变量。
- 对竞争格局、客户验证和份额口径，不要只写成“风险”或“壁垒”；必须说明它们如何影响收入、毛利率、费用率、客户粘性和订单稳定性。
- 核心变量段落必须落到“成立时利润如何变，不成立时逻辑降级成什么”。
- 表格只能做索引、对比和数字承载；关键推理必须用段落讲清楚。
- `final_report.md` 不放独立估值章节，不写目标价、目标市值、PE/PEG/SOTP、买卖建议或“当前贵不贵”的正式估值结论。估值输入留在 `valuation_handoff.md`。

## 必须避免

- 只罗列公告和财务数字。
- 用“平台化、生态化、国产替代、AI 算力、第二曲线”这类词替代解释。
- 看到扩产或募投就写成业绩必然兑现。
- 看到客户合作或认证线索就直接写成订单和利润。
- 只写“竞争格局好、客户优质、认证壁垒高”，但不说明客户验证阶段、替代风险和份额变化。
- 把权益法投资收益、非经常性损益和经营利润混在一起。
- 用“公告未披露，所以无法确认”结束核心变量段落。
- 把市场口径全部塞进证据台账、脚注或反方观点，导致正文只剩低弹性事实。

## 推荐章节

### 1. 一句话结论

说明公司属于预期型、半兑现型、兑现型，还是仍停留在主题映射/旧业务修复。结论必须同时包含：

- 主线为什么成立。
- 最核心的上行变量是什么。
- 哪些经营信号会让主线升级或降级。

### 2. 这家公司到底靠什么赚钱

用普通语言解释主营业务：

- 哪些是旧业务。
- 哪些是新业务。
- 哪些业务毛利率高。
- 哪些业务收入大但利润率一般。
- 哪些利润来自主业，哪些来自投资收益、权益法或非经常性项目。

### 3. 市场为什么重新定价

解释新周期和市场变量：

- AI、数据中心、新能源、机器人、先进制造或半导体等需求变化。
- 公司为什么可能从旧身份切到新身份。
- 这个身份变化会抬收入、抬利润率、抬客户粘性，还是只抬题材热度。
- 市场正在交易的 3 到 8 个变量是什么，每个变量如何影响利润斜率。

### 4. 旧业务给了什么经营下限

不要只说“底盘稳”。要解释：

- 收入是否稳定。
- 客户是否稳定。
- 现金流是否支持。
- 毛利率、费用率、应收、存货和回款质量是否健康。
- 如果新业务不兑现，公司逻辑会降级到哪一档。

### 5. 第二成长曲线给了什么上限

不要把行业空间直接等同于公司利润。要解释：

- 公司在产业链里处于什么位置。
- 价值量、ASP、份额或客户粘性是否提升。
- 市占率、客户导入或订单是否有证据。
- 新业务是抬收入、抬毛利率、抬市场身份，还是只抬市场热度。

### 6. 平台复用为什么让跨界可信

平台复用必须拆开写：

- 技术、工艺、客户、产能、供应链、认证体系和组织能力哪些能复用。
- 哪些能力需要重新建设。
- 复用如何降低客户导入难度、费用率、良率爬坡或交付风险。
- 是否存在硬跨界、能力断层或概念包装。

### 7. 竞争/客户验证链是否支持第二曲线

必须写清：

- 主要竞争对手是谁，它们的产品、客户、产能、成本和认证状态如何。
- 公司客户验证处于接触、送样、认证、小批、量产还是份额提升。
- 客户或份额若只是市场口径，如何降级保留为影响因子和跟踪变量。
- 公司优势来自技术、成本、交付、客户关系、认证周期、产能还是平台复用。
- 替代风险如何让逻辑降级：客户双供/多供、低价抢单、技术路线替代、竞争对手扩产或良率追赶。

### 8. 承接动作如何进入经营兑现

必须区分：

```text
融资/扩产 -> 设备/人员/研发 -> 客户验证 -> 订单/量产 -> 收入确认 -> 利润释放 -> 现金回收
```

每一环说明证据强弱。没有数据的环节要写成可验证缺口和情景边界：若后续被验证，利润斜率怎样变陡；若长期缺失，主线降级到哪一档。

### 9. 利润中枢和利润斜率

必须把利润来源拆开：

```text
后续利润中枢 =
旧业务底盘利润
+ 第二曲线新增利润
+ 平台复用带来的效率收益
- 扩张费用/折旧/财务成本/爬坡损耗
```

如果存在权益法收益、投资收益、少数股东权益或股权资产，必须单独拆出，不要让读者误以为全部利润来自主业。本 skill 默认写轻量利润中枢和敏感变量，不替代正式估值。

### 10. 跟踪体系

写清未来 2 到 4 个季度看什么：

- 订单、收入、毛利率、现金流、应收存货、扩产进度、客户导入、产品放量。
- 哪些信号说明逻辑增强。
- 哪些信号说明只是旧业务修复或概念映射。
- 哪些信号说明主线证伪。

### 11. 反方观点和最终判断

至少写 3 个反方点：

- 最大业务风险。
- 最大财务风险。
- 最大市场变量误判风险。

每个风险后面写跟踪指标。最后判断不要只写“看好/谨慎”，要写“什么条件下成立，什么信号出现会证伪”。

## 市场变量段落写法

高重要变量必须用以下结构进入正文：

```text
市场口径说什么：
证据等级：
为什么市场交易它：
如果成立：
如果不成立：
验证指标：
终稿保留方式：
```

推荐表达：

```text
市场正在押客户导入和高毛利产品放量；若成立，利润桥通过收入增量、毛利率提升和费用摊薄变陡。当前公告只验证了产能、研发或客户合作的一部分，市场口径仍需用订单、分业务毛利率和现金流验证；若长期没有经营兑现，逻辑降级为旧业务修复加概念映射。
```

不推荐表达：

```text
公司未披露客户份额和毛利率，因此无法确认，后续关注。
```

## 语言要求

- 少用“显著、持续、充分、确定性”等强词，除非证据很硬。
- 对普通投资人不熟悉的词，第一次出现时用一句话解释。
- 表格后必须解释含义，不能让表格替代正文推理。
- 结论要有层次：事实、市场口径、推断、假设分开写。
- 章节标题尽量带判断和变量，不要只写栏目名。
- 终稿提纲前必须通过中期结构 QA；若缺竞争客户链、利润斜率或跟踪体系，先补中间文件再写终稿。
- 中期结构 QA 和反方观点不能只列风险，必须保留“乐观派押什么、成立时利润怎样变、什么信号确认升级、什么信号触发降级”。
- 终稿前搜索“仍需验证、证据不足、后续关注、不能确认”等词；若出现在核心变量段落，必须改成变量增强/减弱时结论如何变化。
