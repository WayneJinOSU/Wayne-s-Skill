# 连贯报告写作规范

## 写作目标

终稿要像一篇完整行业研究，而不是各 agent 摘要合集。

读者读完应理解：

- 这个行业到底是什么生意。
- 需求和供给为什么改变价格与利润。
- 利润池留在哪个环节。
- 哪些公司能把行业变化兑现进财报。
- 当前市场定价是否已经反映预期。
- 什么指标会证明研究逻辑错了。

## 主线优先

终稿第一屏必须给出主线：

```text
这个行业处在什么阶段，利润流向哪里，投资判断的核心变量是什么。
```

之后所有章节都服务这条主线。不要把资料按来源或 agent 分段堆放。

## 推荐章节

1. 一句话结论。
2. 先讲人话：行业卖什么，谁买单。
3. 行业边界与产业链。
4. 需求：增长来自哪里。
5. 供给与价格：利润会不会被扩产吃掉。
6. 利润池与竞争格局。
7. 财务映射与重点公司。
8. 估值和市场预期。
9. 风险、证伪点和跟踪指标。

## 表格规则

应保留的表：

- 产业链结构表。
- 价值链利润池表。
- 行业变量到财务指标映射表。
- 跟踪指标表。
- 证伪点表。

不应堆砌的表：

- 大量公司列表。
- 大量无解释的价格历史。
- 只展示 TAM 而不说明公司可获得份额的空间表。

每张表后必须写一句“这张表说明什么”。

## 语言要求

- 第一次出现术语时用人话解释。
- 少用“广阔空间、充分受益、确定性强、平台化”等空词。
- 结论写成条件句，不把情景假设写成事实。
- 不把行业好直接写成股票好。
- 不把短期景气直接写成长期成长。

## 反方嵌入方式

反方观点不要只放最后。关键章节中也要穿插反证：

- 讲需求时，写需求可能不及预期的信号。
- 讲供给时，写扩产或库存的反向风险。
- 讲利润池时，写上下游挤压和替代风险。
- 讲估值时，写当前价格隐含的乐观假设。

最后再集中列出 3 到 7 个最重要证伪点。

## 禁止行为

- 直接拼接各 agent 的小结。
- 只写行业资料，不落到财务传导。
- 只写龙头公司优点，不比较产业链环节利润。
- 只说“估值合理/偏贵”，不解释市场定价到哪一步。
- 把市场预测、券商口径或传闻写成事实。
