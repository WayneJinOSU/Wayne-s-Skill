# 订单出货领先指标研究

## Purpose

用于回答：订单和出货是否真实放量，能否传导到收入、利润和现金流，并提前验证财报尚未完全体现的增长？

这个子 skill 服务于供应链平台型成长股研究主控流程，尤其可被 `$supply-chain-agentic-research` 调度。它只判断“订单出货和增长质量”，不要替代周期、供应链地位、利润桥和估值判断。

## Inputs

优先读取：

- 公司公告、订单、合同、中标、客户项目、量产和出货表述。
- 排产、产能利用率、海关、产业链跟踪、渠道或第三方数据。
- 收入、毛利率、净利率、存货、应收、合同负债、经营现金流。
- 产能扩张、设备采购、在建工程、海外基地、人员和研发投入。
- 可变信息必须核对日期、周期和口径。

## Workflow

1. 汇总订单和出货证据，并按强、中、弱信号分级。
2. 建立传导链：订单 -> 排产 -> 出货 -> 收入确认 -> 利润释放 -> 现金回收。
3. 判断订单是新增长、补库存、集中交付，还是一次性大单。
4. 检查收入质量：是否伴随毛利率、净利率改善，还是只增收不增利。
5. 检查营运质量：应收、存货、合同负债、经营现金流是否匹配增长。
6. 检查产能承接：现有产能、扩产进度、良率和海外基地是否足够。
7. 判断财报滞后程度：订单是否已在收入利润中体现，还是仍有确认空间。
8. 列出订单逻辑失效的反证条件。

## Signal Levels

| 信号等级 | 例子 | 判断方式 |
| --- | --- | --- |
| 强信号 | 明确订单金额/数量、量产、出货放量、财报利润验证 | 可作为主线证据 |
| 中信号 | 扩产、设备、客户认证、合作框架、产能爬坡 | 只能说明承接，需后续验证 |
| 弱信号 | 概念、传闻、笼统进入供应链、未量化表述 | 不能单独支撑结论 |

## Quality Checks

重点检查：

- 收入增长是否快于或慢于订单表述。
- 毛利率是否因产品升级、规模效应或良率改善而提升。
- 存货增长是否合理，是否存在积压风险。
- 应收账款是否异常扩张，回款是否拖累现金流。
- 经营现金流是否长期弱于利润。
- 合同负债或预收是否支持订单可见度。
- 产能扩张是否可能带来折旧和爬坡损耗。

## Discipline

不要把“扩产”当作订单，不要把“订单”直接当作利润，不要把“收入增长”直接当作高质量增长。

供应链平台股的关键是：订单放量必须最终体现为利润中枢上移，并且不能靠明显恶化的应收、存货和现金流换增长。
