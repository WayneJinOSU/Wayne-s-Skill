# Agentic 编排细则

## 总控职责

总控像投研项目经理，不是把每个角色的内容粘起来。

总控必须完成：

- 提出主线假设。
- 判断行业类型和研究边界。
- 建立中间产物目录。
- 调度数据工具和研究角色。
- 处理角色之间的冲突。
- 决定最终报告的叙事顺序。

## 推荐文件结构

```text
research_artifacts/<行业或公司>/
  <行业或公司>_question.md
  <行业或公司>_evidence_index.md
  <行业或公司>_industry_chain.md
  <行业或公司>_demand_supply_price.md
  <行业或公司>_profit_pool_competition.md
  <行业或公司>_financial_mapping_companies.md
  <行业或公司>_skeptic_review.md
  <行业或公司>_report_outline.md
  <行业或公司>_final_report.md
```

## 角色分工

正式研究必须触发 5 个角色。若当前环境支持 subagent，且用户显式要求 subagent、agentic、多角色或并行研究，必须使用真实 subagent；若无法使用真实 subagent，必须用阶段文件模拟。

### 产业链与行业定义 agent

输出文件：`<行业或公司>_industry_chain.md`

任务边界：

- 行业边界和行业类型分诊。
- 产业链上中下游和终端需求。
- 公司或重点环节所处位置。
- 初步判断利润和风险集中在哪些环节。

输出结构：

```text
行业定义：
行业类型：
产业链结构：
公司/重点环节位置：
利润与风险初判：
证据 -> 推理 -> 反证 -> 判断：
对主线的贡献：
可能推翻主线的证据：
终稿中应该放在哪一章：
```

- 供给扩张、产能建设周期、退出难度、库存。
- 价格形成机制和价格趋势。

- 重点公司差异：龙头、成本领先者、技术突破者、概念公司。

### 反方审查 agent

输出文件：`<行业或公司>_skeptic_review.md`

任务边界：

- 只挑错，不负责写漂亮。
- 检查证据链跳步、供需误判、利润池误判、估值误导、技术路线和政策风险。
- 检查普通投资人是否能理解。

输出结构：

```text
最大漏洞：
证据不足的结论：
可能误导的估值或行业口径：
替代解释：
读者理解断点：
必须补充或降级的判断：
最关键 3-7 个证伪点：
```

## 无 Subagent 时的模拟方式

如果当前环境不能启动 subagent，不要退回一次性写作。必须按以下方式模拟：

1. 写 `<行业或公司>_industry_chain.md`。
2. 读取前文件，写 `<行业或公司>_demand_supply_price.md`。
3. 读取前两文件，写 `<行业或公司>_profit_pool_competition.md`。
4. 读取前三文件，写 `<行业或公司>_financial_mapping_companies.md`。
5. 读取前四文件，写 `<行业或公司>_skeptic_review.md`。
6. 读取全部文件，写提纲和终稿。

## 冲突处理

当角色结论冲突时，总控必须显式处理：

- 需求强但供给扩张更快：不能写行业景气，改写为“需求增长但利润被供给稀释”。
- 利润池在上游但公司处在中游：不能把行业利润直接映射到公司。
- 格局稳定但估值已高：结论写“好行业但预期已反映”。
- 技术路线存在替代风险：降低长期确定性，列为核心证伪点。
- 数据口径冲突：官方/公告优先，市场口径作为交易情绪或预期。

## 最终整合

终稿不是拼接五份中间稿。总控必须重新排序：

- 先解释行业卖什么、谁买单。
- 再解释需求和供给如何影响价格。
- 再解释利润为什么留在某个环节。
- 再解释哪些公司能把行业变量兑现进财报。
- 最后讨论估值和证伪。

每个中间稿的“对主线的贡献”只能作为素材，不能原样堆进终稿。
