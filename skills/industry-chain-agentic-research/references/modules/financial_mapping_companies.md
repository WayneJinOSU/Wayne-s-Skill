# 财务映射与重点公司模块手册

## 模块目标

把行业变量落到上市公司财报，区分收入弹性、利润弹性、现金流质量和估值弹性。

```text
行业变量 -> 销量/价格/结构/成本/产能利用率
-> 收入/毛利率/费用率 -> 利润 -> 现金流/资产负债表 -> 估值
```

## 核心问题

- 哪些行业变量抬收入，哪些抬毛利率，哪些只抬资本开支或费用。
- 公司收入确认是否领先、同步或滞后于行业景气。
- 毛利率改善来自价格、产品结构、规模效应、成本下降，还是会计/一次性因素。
- 增长是否消耗应收、存货、资本开支和自由现金流。
- 哪些公司是真兑现，哪些是半兑现，哪些只是概念映射。
- 行业变量进入重点公司的路径是否一致，还是每家公司都有不同瓶颈。

## 必须查的数据

- 收入、毛利率、净利率、费用率、经营现金流、自由现金流。
- 应收账款、票据、存货、合同负债、预付款、资本开支、折旧。
- 分业务收入和毛利率、客户结构、海外收入、产品结构。
- 订单、排产、产能利用率、募投/扩产、在建工程。
- 同行对比：谁收入快、谁利润好、谁现金流差、谁估值更高。

## 禁止跳步/误区

- 不得把行业景气直接写成公司利润。
- 不得把订单直接写成收入，必须说明交付和收入确认。
- 不得把收入增长直接写成利润平台上移。
- 不得只看营收增速或净利润，忽略毛利率、一次性收益和现金流。
- 不得忽略应收、存货、合同负债、折旧和资本开支。
- 不得只列公司名单，必须分层说明财务兑现质量。

## 专属输出字段

除 `orchestration.md` 的通用字段外，本模块必须输出：

```text
财务传导链：
行业变量 -> 财务指标映射表：
重点公司分层：
收入弹性与利润弹性差异：
现金流和资产负债表风险：
概念公司剔除标准：
```

## 公司分层

- 兑现型：收入、利润、毛利率、现金流和订单证据较一致。
- 半兑现型：收入或订单已出现，但利润率/现金流仍待验证。
- 预期型：产业位置合理，但财务尚未兑现。
- 概念型：缺少订单、客户、收入或现金流证据。

## 判断强度标准

- 强：行业变量、订单/客户、收入、利润率、现金流形成闭环。
- 中强：收入和利润兑现较好，但现金流或资产负债表有压力。
- 中：收入兑现，但利润率和现金流证据不足。
- 弱：只有概念或客户传闻，没有财务验证。

## 证伪指标

- 收入增长但毛利率下滑。
- 应收和存货增速显著高于收入。
- 合同负债下降或订单转弱。
- 经营现金流持续弱于净利润。
- 扩产后产能利用率不足，折旧吞噬利润。
