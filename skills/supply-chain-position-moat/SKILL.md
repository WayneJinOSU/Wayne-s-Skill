# 供应链地位与壁垒研究

## Purpose

用于回答：公司在产业链中是否处于关键位置，壁垒来自哪里，是否具备阶段性不可替代性和合理议价能力？

这个子 skill 服务于供应链平台型成长股研究，通常被 `$supply-chain-agentic-research` 吸收为“供应链地位与壁垒”模块。它只判断“位置和壁垒”，不要替代周期、订单出货、利润桥和估值判断。

## Inputs

优先读取：

- 公司客户结构、认证资质、合作年限、重点客户项目。
- 产品复杂度、良率、交付周期、产能规模、全球基地。
- 同行业竞争格局、份额、扩产、替代供应商。
- 毛利率、净利率、价格趋势和客户集中度。
- 公司公告、年报、投资者关系纪要、官网和可靠产业数据。

## Workflow

1. 定义产业链角色：核心技术定义者、关键零部件/材料、平台型制造交付、普通代工或边缘配套。
2. 判断客户重要性：是否进入头部客户核心供应链，是否有多客户扩散。
3. 拆壁垒来源：认证、规模、交付、良率、成本、产能、供应链组织、全球布局、工程响应。
4. 判断可替代性：替代难度、替代周期、竞争对手能力、客户双供/多供策略。
5. 判断议价权：公司能否分享价值量提升，还是只承担放量但被压价。
6. 检查客户集中风险：单一客户或单一平台变化是否会显著影响收入利润。
7. 给出地位等级：核心平台、关键供应商、重要配套、普通供应商、证据不足。
8. 列出会削弱地位判断的反证。

## Position Levels

| 等级 | 特征 | 估值含义 |
| --- | --- | --- |
| 核心平台 | 多客户、多产品、复杂交付、难替代 | 可获得身份重估 |
| 关键供应商 | 单一环节强壁垒，客户认证较深 | 有重估弹性但受环节限制 |
| 重要配套 | 有订单和产能，但替代者较多 | 主要看利润兑现 |
| 普通供应商 | 产品同质、客户压价、替代容易 | 不宜给平台溢价 |

## Discipline

不能因为公司不是最高技术定义者就直接否定它；供应链平台公司的壁垒可以来自客户认证、规模交付、良率和全球组织能力。

也不能把“大客户供应商”自动等同于“平台型公司”。必须验证替代难度、利润率和多产品/多客户扩散。
