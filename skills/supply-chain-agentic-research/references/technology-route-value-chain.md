# 技术路线到价值量链路

用于技术路线、产品代际或终端平台快速变化的供应链公司。目标是避免只写“景气度提升、国产替代、高端化”，而漏掉市场真正交易的边际变量。

## 通用链路

研究时按以下链路拆解：

```text
下游终端或平台迭代
-> 性能瓶颈变化
-> 零部件/材料规格升级
-> 单机价值量、ASP、毛利率或用量变化
-> 供应商认证、份额、良率、产能约束
-> 订单、收入、利润和现金流兑现
```

每一层都要区分事实、市场口径和假设。

## 输出要求

可以先用表格在 evidence_index 或对应模块文件中索引变量：

| 层级 | 当前市场交易的变量 | 证据来源 | 证据等级 | 公司相关性 | 收入/毛利/现金流影响 | 变量增强/减弱时的结论变化 |
| --- | --- | --- | --- | --- | --- | --- |
| 下游平台 |  |  |  |  |  |  |
| 性能瓶颈 |  |  |  |  |  |  |
| 规格升级 |  |  |  |  |  |  |
| 价值量变化 |  |  |  |  |  |  |
| 供应链认证/份额 |  |  |  |  |  |  |
| 财务兑现 |  |  |  |  |  |  |

但表格不能替代正文。每个高重要变量必须用段落写清：

```text
变量：市场正在交易什么。
为什么会带来新增需求：下游变化如何改变用量、价值量、规格、认证门槛或供需关系。
公司在哪个环节受益：公司是材料、零部件、设备、制造、系统、渠道，还是控股/参股公司受益。
财务传导路径：影响量、价、毛利率、份额、费用、折旧、良率、少数股东权益、投资收益中的哪几项。
证据属性：公告事实、客户/行业一手资料、市场/券商口径、情景假设或弱线索。
变量增强时：收入、ASP、毛利率、订单稳定性、现金流或利润中枢会怎样变化。
变量减弱时：逻辑会降级成什么，是普通周期修复、补库涨价、低端放量、扩产先行、成本挤压，还是主线证伪。
```

核心变量段落的结尾必须从证据边界转化为判断边界：如果变量成立，利润怎么变陡；如果变量不成立，逻辑降级成什么。

## 市场变量处理

对下列变量，即使没有公告充分确认，也要降级保留：

- 终端平台或产品代际切换时间。
- 单机价值量、ASP、用量、毛利率变化。
- 客户认证、供应商清单、份额、良率、排产。
- 高端产品收入占比和产能利用率。
- 关键原料、设备或产能瓶颈。

写法：

- “公告事实”：公司或客户已经披露，可支撑主结论。
- “市场口径”：券商、产业链或专家资料，多源交叉后可作为情景变量。
- “情景假设”：用于利润桥和敏感性，不写成确定事实。
- “弱线索”：仅作为跟踪点，不支撑主结论。

## 利润桥敏感性

至少选择 3 个关键变量做影响预估和敏感性分析。不得只给数字，必须解释影响逻辑：

- 产品代际或平台放量提前/延后 1-2 个季度。
- 高端产品占比提升或下降。
- ASP 或毛利率变化。
- 客户份额或认证进度变化。
- 良率、爬坡速度、产能利用率变化。
- 关键原料或设备短缺造成的交付限制。

### 价格与利润传导

技术路线升级型公司必须单独拆价格和利润传导，不能只写“高端化”。至少回答：

```text
上一代产品是什么，价格/价值量/毛利率大致处在什么区间。
新一代产品是什么，价格/价值量/毛利率为什么更高。
价差来自性能、认证、良率、稀缺、客户粘性，还是短期供需。
公司能拿到多少：公司事实、市场口径和情景假设分开。
产品占比提升 5/10/20 个百分点，对收入和毛利有什么影响。
原材料、良率爬坡、折旧、费用和少数股东损益如何扣减。
```

可接受写法：

- “若只是普通产品放量，收入会涨但利润弹性有限。”
- “若新一代产品规模量产，同样出货面积会带来更高 ASP 和更高毛利。”
- “若原材料涨价能转嫁，是利润放大器；若客户压价或需求降温，就是利润挤压器。”

不可接受写法：

- “高端化带来利润弹性，后续关注占比。”
- “市场口径仍需验证。”
- “公司有望受益于国产替代。”

## 终稿 QA

写终稿前先从中间文件提取 5-10 个“市场真正交易的变量”，再检查终稿是否保留。若删除，必须说明：

- 证据强度较低但已在反方或情景变量中保留。
- 与公司相关性弱。
- 与其他变量合并表达。
- 已被公告、财报或竞品资料反证。

终稿还必须搜索并处理以下词：

```text
后续关注、仍需验证、不能写成事实、证据不足、待验证、高端化、国产替代、结构升级、平台型
```

若这些词出现在核心变量段落，必须改写为价格、价值量、毛利率、良率、份额、现金流、客户替换成本或逻辑降级的解释。
