# Research Posture

本文件承接 `SKILL.md` 的研究姿态要求。正式研究必须先建立市场交易主线，再做证据分层；证据边界服务于判断，不替代判断。

## Core Posture

- 先回答市场正在重新定价什么，再回答哪些证据已经兑现、哪些仍是市场口径。
- 公告事实确定底线：收入、利润、现金流、产能、项目进展、官方披露产品和客户边界。
- 市场口径打开弹性：产品代际、客户份额、订单排产、ASP、毛利率、单机/单柜/单车/单套价值量、关键原料、良率、供应商清单。
- 情景假设负责天花板：用可调整的数字或区间展示“如果市场口径成立，利润中枢怎样抬高；如果不成立，回到哪一档”。
- QA 负责校准和证伪，不负责把终稿改成审计底稿或风险列表。

## Aggressive Spine Rule

终稿必须先建立进攻型主线。写作时优先回答：

```text
市场正在交易什么？
这家公司为什么不只是普通周期修复？
哪些变量会让利润斜率变陡，而不是只让收入线性增长？
公告事实证明了哪一段，市场口径打开哪一段天花板，下一季看什么来确认？
```

高重要但未公告验证的变量，例如产品代际升级、终端平台迭代、客户份额、订单排产、关键原料锁定、瓶颈产能、单机/单柜/单车/单套价值量、ASP、毛利率或良率，不得只写成“待验证风险”。必须写成“市场押注 -> 利润传导 -> 成立时升级 -> 不成立时降级 -> 验证指标”。

## Section Title And Judgment Rule

章节标题必须尽量包含“变量 + 机制 + 结论方向”，让读者只看目录就能知道研究者在押什么、利润桥怎么变、跟踪点在哪里。

强烈避免以下标题句式：

```text
不是 A，而是 B
不只是 A，也可能 B
A 仍需验证
A 有望受益
A 代表公司下注
技术路线分析
竞争格局梳理
```

应改成更有交易含义的结论型标题：

```text
下一代产品渗透率决定 ASP 斜率，验证点是客户认证和分部毛利率
关键原料紧缺把涨价从成本压力变成高端交付权重
近端扩产贡献未来 4-8 季度供给弹性，远期基地决定下一轮高端产能天花板
海外龙头锁定最高端份额是核心反证，公司必须用认证、份额和高端毛利率突破
```

标题不要求预测一定兑现，但必须写清“如果兑现，什么财务变量会变”。如果标题只表达态度、框架或谨慎程度，而没有产品代际、ASP、毛利率、份额、产能、现金流、少数股东或验证指标中的至少一个，应重写。

## Evidence Grading Timing Rule

证据等级应嵌入变量段落，而不是在段首不断刹车。推荐写法：

```text
市场正在押 X；若 X 成立，利润桥通过 A/B/C 变陡；目前公告已验证 D，产业链口径指向 E，尚需 F 验证。
```

不推荐写法：

```text
X 尚未公告验证，因此只能谨慎看待。
```

## Market Variable Writing Rule

对所有高重要市场变量，终稿必须采用以下写法：

```text
市场口径说什么：
证据等级是什么：
兑现阶段是什么：
为什么市场交易它：
如果成立：
如果不成立：
如何验证：
```

变量增强时，必须写清收入、ASP、毛利率、份额、客户替换成本、良率、现金流或归母利润会怎样变化。变量减弱时，必须写清逻辑降级成普通周期修复、补库涨价、低端放量、扩产先行、成本挤压，还是主线证伪。

不合格写法：

```text
公司切入下一代产品，高端化带来利润弹性，后续关注客户认证。
原材料涨价可能推升行业景气，但公司转嫁能力仍需验证。
公司扩产代表对高端周期的下注，未来有望贡献利润。
客户份额和排产尚未公告披露，因此只能作为风险提示。
```

合格写法：

```text
市场正在押下一代产品渗透率提升；若公司进入核心客户认证并放量，利润桥会通过单件价值量、ASP 和高端毛利率变陡。当前公告只验证了产品和产能边界，客户份额仍是市场口径；下一步看分产品收入、毛利率、订单和现金流。若认证或份额不兑现，逻辑降级为普通产品放量或周期修复。
```

```text
关键原料涨价不是天然利好。若高端产品稀缺、客户接受顺价且公司库存和交付能力充足，涨价可能通过 ASP 留存和毛利率稳定放大利润；若需求降温或客户压价，涨价会变成现金流和毛利率压力。验证指标是调价公告、分产品毛利率、存货周转、应收和经营现金流。
```

```text
扩产只有在客户认证、排产、良率和产能利用率同步提升时才会变成利润斜率；若扩产领先订单，先进入报表的是折旧、人员、研发、爬坡损耗和营运资金占用。验证指标是订单/排产、在建工程转固、产能利用率、毛利率和现金流。
```

## Reference-Grade Report Rule

优秀参考报告的核心不是“把所有风险都说一遍”，而是让读者看完后知道市场为什么愿意交易这家公司、未来利润桥在哪里变陡、什么信号会让逻辑升级。默认按参考报告级别写作：

1. 先给主笔判断：公司发生了什么身份切换，市场错在哪里，利润桥为什么会变。
2. 再给产业链机制：终端平台/技术路线如何传导到价值量、ASP、毛利率、份额和现金流。
3. 再给经营承接：订单、产能、客户认证、原料、良率和子公司如何把预期变成报表。
4. 再给三情景：底线不是保守废话，基准不是中性描述，上行必须有明确触发器。
5. 最后给反方：反方是用来定义证伪条件，不是用来削弱主线气势。
