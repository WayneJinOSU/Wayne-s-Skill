## 3. 日常操作 SOP

### 3.1 调仓日操作(每季度 1 次,15 分钟)

```bash
# 前一交易日 17:00 之后(数据已结算)
cd /home/mira/files/L2_ETF_LIVE
python3 quarterly_picks.py
# → 生成 picks_YYYY-MM-DD.csv (新清单)
# → 生成 trades_YYYY-MM-DD.csv (与上次持仓的 diff:卖单+买单)
```

注意:`quarterly_picks.py` 不内置行情刷新器。若要更新数据,先在外部刷新 `kline_cache_all`,再运行本脚本;`--refresh-data` 只会明确报错提醒,避免误以为已经刷新。

### 3.2 调仓日下单(开盘后 10 分钟,~10 分钟)

1. **09:40** 打开券商 App,先看 `trades_*.csv` 的 SELL 段:把当前持有但本期淘汰的 ETF 全部卖出
2. **等卖单成交完**(通常 1-3 分钟),拿到现金
3. **按 `trades_*.csv` 的 BUY 段下单**:对每只新买入或加仓的 ETF,用**当时盘口价 +0.5% 限价单**
4. **全部成交后**,把 `current_holdings.json` 更新(或让脚本自动写)

### 3.3 季度中(非调仓日)

- ⛔ 不看新闻调仓
- ⛔ 不止盈止损
- ✅ 每周末记一次净值到 `nav_log.csv`(可手动,可自动)
- ✅ 出现 ETF 停牌/退市/分红除权 → 不动作(脚本会在下次调仓时自动处理)

## 4. 风险预警与停止条件

触发以下任一条件立即**暂停下次调仓 + 重新评估**:

1. **单季 drawdown < -15%**(回测最大单季 -16%,超出即异常)
2. **累计相对 MARKET_ADV5M 落后 > 30pp**(策略失效信号)
3. **池子规模缩到 < 80 只**(ADV50M 档,流动性恶化)
4. **HS300 单季 -20% 以上**(类比 2022-Q1 / 2024-01 的极端环境)
