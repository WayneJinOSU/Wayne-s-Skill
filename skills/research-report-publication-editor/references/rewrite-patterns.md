# 改写模式

## 删除型

```text
所用 skill：supply-chain-agentic-research
-> 删除

subagent 中间产物
-> 删除；若需要保留，改为“内部研究底稿”，但不得作为对外来源。
```

## 内部流程 -> 报告判断

```text
终稿应把服务器放在“需求兑现层”
-> 服务器更适合被定位为需求兑现层，而非核心利润池。

终稿必须把这个逻辑拆清楚，不能用“行业景气”一句话替代公司订单。
-> 行业景气只能提供需求底座，北方华创的 alpha 仍需由订单、份额和现金流证明。

不能写成强投资判断。
-> 当前证据尚不足以支撑强判断，应保留为观察线或情景变量。

正式报告必须写清楚...
-> 本节需要回答的核心问题是...
```

## 写作指令 -> 研究判断

判断标准：主语是“终稿/报告/写法/章节/结论/执行纪律”时，多数是内部写作指令；主语换成“经营逻辑/利润桥/证据边界/市场定价/变量传导”后，才能进入正式报告。

```text
报告不能把行业 beta 直接写成公司 alpha。
-> 行业 beta 只提供需求底座，公司 alpha 仍需由订单、份额和现金流验证。

正确写法是分层保留。
-> 这些变量应按证据等级分层保留。

软件章节必须用“收入质量”而不是“产品能力”判断。
-> 软件环节的核心验证指标是收入质量，而非单纯的产品功能发布。

最终执行纪律是先反方、再上修。
-> 结论形成遵循“先识别反方约束、再评估上修条件”的路径。

反方也不能把报告压成保守摘要。
-> 反方证据约束利润外推，但并不否定收入底盘和平台化证据。
```

## 强约束词用途判断

“必须/不能/只能/应当”不是禁词。判断它是否需要改写，先看它约束的对象。

允许保留：

```text
不能把未披露订单金额写成事实。
-> 保留。这是证据边界。

合同负债不能等同于完整订单。
-> 保留。这是财务科目边界。

单季现金流改善不能直接外推为长期拐点。
-> 保留。这是风险约束。

毛利率修复必须由产品结构、材料费和费用率共同验证。
-> 保留。这是利润桥硬约束。
```

需要改写：

```text
报告不能写成强投资判断。
-> 当前证据尚不足以支撑强投资判断。

结论不能写成全面替代海外龙头。
-> 当前更适合定义为在客户多供体系中扩大工艺段和采购份额。

终稿必须处理合同负债下降。
-> 合同负债下降构成订单可见度的黄灯，需要进入后续跟踪。

最终执行纪律是继续观察。
-> 后续跟踪重点是合同负债、现金流、毛利率和研发费用率能否同向改善。
```

## 提示词口吻 -> 证据边界

```text
只能写成行业硬约束。
-> 目前更适合作为行业硬约束，而非单家公司利润兑现证据。

不能支撑正文强结论。
-> 该证据只能支撑中弱判断，尚不足以进入强主线。

必须拆成两面。
-> 该变量对供应商和需求方形成两条不同传导链。

软件章节必须用“收入质量”而不是“产品能力”判断。
-> 软件环节的核心验证指标是收入质量，而非单纯的产品功能发布。

利润桥不能用收入外推。
-> 利润桥需要从收入、毛利率、费用、折旧、少数股东和现金流逐层验证。
```
