# 需求/供给/价格模块手册

## 模块目标

判断行业景气是否能转化为价格、利润和订单，而不是只证明“需求在增长”。

```text
需求增速 - 供给增速 - 库存压力 -> 价格方向 -> 利润弹性
```

## 核心问题

- 需求来自新增、替换、渗透率、客户 capex、政策、出口、补库存，还是价格刺激。
- 需求是一次性拉动、周期性复苏，还是长期渗透率提升。
- 需求证据来自真实销量/订单/招标/客户 capex，还是只来自市场叙事。
- 终端需求到研究环节有几层传导，每层是否会削弱弹性。
- 供给扩张周期多长，瓶颈在资金、设备、良率、认证、牌照、资源还是人才。
- 现有库存、在建产能、潜在进入者和替代产能是否足以消化需求。
- 价格由现货、长协、招投标、成本加成、渠道、品牌、政策，还是客户年度降价决定。
- 行业变量是抬价格、抬销量、抬产品结构，还是只抬资本开支。

## 必须查的数据

- 官方/协会：终端销量、装机、渗透率、出口、库存、开工率。
- 公司/客户：客户 capex、订单、招标、长协、产能、扩产公告、收入确认。
- 价格口径：现货/期货、招投标价、ASP、渠道批价、合同价。
- 财务交叉验证：毛利率、存货、合同负债、应收、经营现金流。
- 高频信号：排产、交付周期、库存天数、开工率、渠道动销。

## 禁止跳步/误区

- 不得把 TAM 大直接写成需求确定。
- 不得把下游 capex 增长直接写成公司订单增长。
- 不得只写需求，不写供给扩张和库存。
- 不得把补库存当成终端需求改善。
- 不得把价格上涨外推为长期趋势，除非供给约束可持续。
- 需求强但价格弱、需求强但供给更强时，必须降级景气结论。

## 专属输出字段

除 `orchestration.md` 的通用字段外，本模块必须输出：

```text
需求结论：
供给结论：
价格机制：
当前景气阶段：
需求证据表：
供给和库存证据表：
价格证据表：
需求 -> 供给 -> 价格 -> 利润传导：
```

## 判断强度标准

- 强：终端需求、客户订单/招标、价格或毛利率、库存/供给约束同时支持。
- 中强：需求和订单较强，但价格或供给约束证据不完整。
- 中：只证明需求增长，供给、价格或库存证据不足。
- 弱：主要来自券商/媒体叙事，缺少销量、订单、价格或财务交叉验证。

## 证伪指标

- 终端销量、客户 capex、订单或招标连续走弱。
- 库存上升、开工率下降、交付周期缩短。
- 新增产能集中释放，价格或 ASP 下行。
- 毛利率没有跟随景气改善，或经营现金流恶化。
