# 数据路径与证据优先级

## 目标

本 skill 不替代 `$tushare`、`$wencai-query`、公告 PDF 处理或外部搜索能力。它负责确定证据链、调用顺序、市场变量扩表和口径解释。正式估值交给 `$growth-stock-valuation`；本文件只说明哪些数据应进入 `valuation_handoff`。

## 证据优先级

1. 官方公告原文：年报、季报、业绩预告/快报、订单、募投、定增、可转债、重组、关联交易、问询回复、业绩说明会。
2. 结构化数据：`$tushare` 的股价、市值、财务三表、财务指标、现金流、股本、分红和历史行情。市值和估值类字段只用于市场阶段判断或 `valuation_handoff`，不在终稿形成正式估值结论。
3. `$wencai-query`：题材概念、市场排名、阶段涨幅、同行筛选、事件性条件和快速交叉验证。
4. 公司官网、投资者关系、交易所互动、客户/竞品公告、可靠新闻和行业资料：用于补充业务进展、客户、订单、产品和产业趋势。
5. 市场增量情报：券商研报核心假设、产业链调研、专家问答、交易热词、新业务客户线索、同行/竞品进展和客户认证线索。此类资料不能写成公告事实，但必须用于识别市场真正交易的边际变量。

公告原文与结构化数据冲突时，以公告原文为准，并说明差异口径。行情、市值或估值类字段若被采集，必须注明查询日期和用途。

## 市场变量分层

底盘型成长股常由“旧业务托底 + 新业务弹性”共同重定价。公告事实确定底线，市场口径识别弹性变量；关键变量要进入主线、利润斜率和跟踪体系，同时标明证据等级。

| 类型 | 例子 | 写法 |
| --- | --- | --- |
| 公告事实 | 财报利润、分业务收入、订单公告、募投产能、管理层明确披露 | 可支撑主结论 |
| 结构化数据 | 股价、市值、财务指标、现金流、应收存货、历史行情 | 支撑财务质量、市场阶段和接力输入 |
| 市场/券商口径 | 客户份额、产品 ASP、毛利率、订单节奏、产业链调研 | 作为影响因子进入主线和情景变量 |
| 合理推断/情景假设 | 根据产能、客户、行业空间和费用率推演的利润斜率 | 标注假设，写清成立/不成立的逻辑降级 |
| 弱线索/传闻 | 无法交叉验证的客户导入、团队变化、排产或份额 | 只作为跟踪变量，不支撑强结论 |

对高重要市场变量，必须用段落写清：

- 市场正在交易什么变量。
- 它影响收入、毛利率、费用率、客户粘性、现金流或归母利润中的哪几项。
- 证据属性是什么：公告事实、结构化数据、市场/券商口径、情景假设或弱线索。
- 变量增强时，利润中枢或利润斜率如何变化。
- 变量减弱时，逻辑降级成什么：旧业务修复、普通扩产、概念映射、费用压力，还是主线证伪。
- 哪个公告、财报项目、订单、分业务毛利率、现金流或客户验证能继续确认。

## 单公司深度研究

默认路径：

```text
1. 形成主线假设：市场为什么重新定价，哪些变量决定利润斜率
2. 读取最新年报/季报/业绩预告/重大事项公告
3. 调用 `$tushare` 拉财务、现金流、历史行情和市值；估值类字段只进入 handoff
4. 调用 `$wencai-query` 查概念、同行、阶段涨幅、市场排名和交易热词
5. 搜集券商口径、产业链线索、客户/竞品资料、客户认证线索和公司官网资料，建立市场变量索引
6. 写 `<标的>_evidence_index.md` 和 `<标的>_market_repricing_spine.md`
7. 分发给旧业务底盘、第二曲线、平台复用、竞争/客户验证链、承接动作、利润桥、跟踪体系和反方审查
8. 写轻量利润中枢、`<标的>_valuation_handoff.md` 和 `<标的>_midterm_structure_review.md`
9. 通过中期结构 QA 后写投资人可读报告；终稿不写目标价、目标市值、PE/PEG/SOTP 或买卖建议
```

## 快速判断

当用户只要快速看一只股票，可以使用轻量模式，并必须说明未启用完整 agentic 流程。

轻量模式至少覆盖：

- 市场真正交易的 3 到 5 个变量。
- 旧业务经营下限。
- 第二成长曲线是否抬收入、抬利润率或抬市场身份。
- 承接动作是否进入订单、收入、利润或现金流验证。
- 客户验证和竞争格局是否支持第二曲线份额与毛利率假设。
- 利润斜率和最敏感变量。

不能把轻量模式输出包装成完整深度报告。

## 横向比较或选股

当用户要求“找一批”“横向比较”“谁更强”：

```text
`$wencai-query`：先筛候选池、题材口径和市场排名
`$tushare`：补财务质量、现金流、市值和行情指标
公告原文：只抽 Top 候选或争议公司的关键公告
市场线索：按新业务客户、竞争格局、份额、产品价值量和承接动作做差异化比较
```

不要对所有候选公司逐一读年报，先用数据缩小范围。

## 重大事项驱动

当研究重点是重组、定增、可转债、订单、扩产、减持或实控人变化：

```text
公告原文优先
`$tushare` 补市值、行情、现金流和财务承受能力
`$wencai-query` 补市场反应、相关概念和同类标的
券商/产业链口径补充市场正在如何解释该事项
```

最终判断必须区分“事实已公告”“公司预计”“市场推演”和“情景假设”。

## 模块对应数据

| 模块 | 优先数据 | 辅助数据 |
| --- | --- | --- |
| 市场重定价主线 | 券商口径、产业链线索、交易热词、阶段涨幅、公告催化 | wencai 概念、同行表现、客户/竞品资料 |
| 旧业务底盘 | 年报/季报、财务三表、现金流、应收存货、客户结构 | 行业对比、同行筛选 |
| 第二成长曲线 | 年报业务描述、募投/扩产公告、行业数据、客户应用场景 | 券商模型、产业链调研、同行空间对比 |
| 平台复用 | 年报产品/客户/供应链、工艺、认证、研发披露 | 公司官网、投资者关系、专利/招投标线索 |
| 竞争/客户验证链 | 同行公告、客户合作、送样/认证/量产、份额和替代风险 | 券商口径、产业链调研、客户/竞品官网、招投标线索 |
| 承接动作 | 募投、定增、可转债、订单、扩产、设备、海外基地 | 现金流、负债、事件筛选、人员/研发变化 |
| 利润桥 | 分业务收入/毛利率、订单、产能、费用率、资本开支 | 行业毛利率、同行净利率、募投效益测算 |
| 跟踪体系 | 财报披露节奏、订单/客户节点、扩产计划、现金流和营运资本 | 市场变量表、管理层指引、行业周期 |
| 中期结构 QA | 各中间文件、市场变量表、模块覆盖表、提纲草案 | 检查缺竞争客户链、利润斜率、跟踪体系和估值接力输入 |
| 估值接力输入 | 市值/行情、事实底盘、利润情景、证伪点 | 一致预期、同行口径、历史区间；正式估值用 `$growth-stock-valuation` |

## 输出口径

每份报告至少说明：

- 使用了哪些数据路径和 skill 路径。
- 最新数据日期和公告报告期。
- 哪些结论来自公告事实，哪些来自市场口径、合理推断或情景假设。
- 哪些市场变量进入正文、利润斜率和跟踪体系。
- 竞争/客户验证链如何影响第二曲线、承接动作和利润斜率。
- 中期结构 QA 是否通过，若未通过补写了哪些内容。
- 哪些输入只进入 `valuation_handoff`，不进入终稿估值结论。
