# 主工作流与子工作流

## 执行原则

每次分析必须由主工作流主导，而不是机械跑完所有子工作流。主工作流先提出研究假设，再选择需要深挖的子工作流，最后根据证据权重合成结论。

子工作流的角色是回答主工作流提出的问题。每个被调用的子工作流都要输出：

- 结论：成立、部分成立、不成立、证据不足
- 核心证据：尽量引用财报、公告、交易所互动、公司官网、投资者关系纪要、可靠行情数据
- 不确定性：哪些信息缺失或可能改变判断
- 对主线判断的影响：提高确信度、降低确信度、提高估值弹性、提高证伪风险

不要平均展开所有子工作流。最终报告只重点呈现最影响判断的 2 到 3 个子工作流，其他子工作流放入调度表即可。

## 主工作流：研究调度

目标：像投研项目经理一样组织分析，而不是把七层框架切段罗列。

### Step 1：提出主线假设

先用一句话判断市场正在交易什么：

- 身份重估：公司从旧业务公司变成 AI/新能源/机器人/先进制造链条公司
- 利润中枢上移：新业务可能把未来利润抬到新的台阶
- 订单兑现：市场在验证订单、产能、客户和量产
- 估值透支：未来几年利润已经被提前定价
- 错杀修复：基本面没坏，但股价过度反映悲观预期

同时提出 3 个核心问题，例如：

- 旧业务能不能托住下限？
- 新业务是不是真能抬高天花板？
- 当前市值是否已经透支 2026-2028 年利润？

### Step 2：框架适用性分诊

快速判断：

- 适用：旧业务底盘、新成长曲线、复用能力、承接动作都基本成立
- 部分适用：有底盘或新曲线，但复用/承接/利润桥有明显缺口
- 不适用：行业逻辑不匹配，或只是概念股、周期股、金融地产、纯消费、早期创新药等

如果不适用，停止完整框架，改为说明为什么不适用。

### Step 3：选择子 skill

默认最小调用：

- `$growth-base-business`：判断旧业务下限
- `$growth-second-curve`：判断新业务上限
- `$growth-forward-valuation`：判断当前是否透支，并输出证伪点

按需调用：

- `$growth-platform-reuse`：当新业务可信度是关键争议时调用
- `$growth-execution-signals`：当市场主要争议是订单、扩产、量产、客户验证时调用
- `$growth-profit-bridge`：当估值能否消化是核心问题时调用

### Step 4：权重排序

把子工作流分成三类：

- 主导结论：决定最终判断的 1 到 2 个子工作流
- 支撑结论：提供辅助证据的 2 到 3 个子工作流
- 风险约束：主要用于提醒证伪风险的子工作流

最终报告必须说明为什么某些子工作流权重更高。

### Step 5：合成最终判断

最终判断回答：

- 公司属于预期型、半兑现型、兑现型，还是不符合框架
- 最大机会来自哪里
- 最大风险来自哪里
- 当前市值是在低估、合理定价，还是透支未来
- 适合什么类型资金

## 子工作流 A：底盘是否够硬

目标：判断公司下限。

检查项：

- 原有业务收入规模和增速
- 原有业务利润和毛利率
- 经营现金流和应收/存货压力
- 客户基础和行业地位
- 业务是否足够分散
- 没有新业务时公司是否仍能维持合理盈利

输出小结：

```text
底盘结论：
核心证据：
主要风险：
对估值的影响：
```

## 子工作流 B：新赛道是否抬高天花板

目标：判断公司上限。

检查项：

- 新业务是什么
- 对应市场空间和未来 3-5 年增速
- 行业处于早期、中期还是成熟期
- 技术路线是否清晰
- 公司处于核心环节还是边缘环节
- 单机价值量或单位价值量是否提升

输出小结：

```text
天花板结论：
新业务：
行业空间：
公司位置：
关键不确定性：
```

## 子工作流 C：平台复用能力

目标：判断可信度。

检查项：

- 技术是否复用
- 工艺是否复用
- 客户是否复用
- 供应链是否复用
- 产能是否复用
- 认证体系是否复用
- 是否存在完全跨界或概念化包装

输出小结：

```text
复用结论：
复用链条：
跨界风险：
可信度：
```

## 子工作流 D：承接动作

目标：判断预期能否落地。

检查项：

- 扩产公告
- 资本开支
- 定增、H 股、可转债、项目贷款
- 海外基地
- 设备采购
- 客户验证或定点
- 订单释放
- 量产进度
- 研发投入和人员扩张

输出小结：

```text
承接结论：
已发生动作：
尚未验证事项：
落地风险：
```

## 子工作流 E：成长阶段

目标：判断市场交易的是预期差、兑现，还是透支。

分类：

- 预期型：新业务刚被发现，利润尚未明显体现
- 半兑现型：新业务已放量，市场验证持续性
- 兑现型：业绩大幅兑现，市场充分定价
- 不符合：没有底盘或没有真实新曲线

检查项：

- 新业务是否已有收入或订单
- 利润是否已经体现
- 股价是否已提前反映
- 市场仍在担心什么
- 财报是否证伪或验证市场担忧

输出小结：

```text
阶段结论：
为什么不是上一阶段：
为什么还没到下一阶段：
股价弹性来源：
```

## 子工作流 F：利润桥

目标：把故事转成利润中枢。

固定公式：

```text
未来利润 =
存量业务利润
+ 新业务新增利润
- 扩张期费用/折旧/财务成本/爬坡损耗
```

拆解要求：

- 存量业务利润：用历史利润、毛利率、费用率建立底盘
- 新业务收入：用 TAM、市占率、价值量、产能或订单估算
- 新业务利润率：考虑毛利率、净利率、规模效应、议价能力
- 扩张期成本：研发、折旧、人员、销售、财务、汇兑、产能爬坡

## 子工作流 G：远期估值

目标：判断当前市值是否透支。

要求：

- 使用最新股价和总市值，并注明日期
- 用 2025、2026E、2027E、2028E 利润假设计算 PE
- 明确利润假设是保守、中性还是乐观

## 子工作流 H：证伪点

目标：找出最可能推翻逻辑的变量。

至少覆盖：

- 新业务订单或客户验证
- 毛利率和净利率
- 产能爬坡和折旧压力
- 行业技术路线变化
- 竞争格局恶化
- 当前估值是否已经透支
- 现金流、应收账款、存货等财务质量

输出要求：

按重要性列 3-5 个证伪点，并说明应该跟踪什么数据。

## 最终报告合成

最终报告不要简单拼接子工作流。需要先给出主线假设和子工作流调度表，再重点展开最影响判断的子工作流：

- 一句话结论
- 主线假设
- 子工作流调度表
- 2 到 3 个核心证据
- 当前成长阶段
- 利润桥和估值表，如果估值是主问题
- 最关键证伪点和跟踪指标
- 对不同类型资金的适配度

建议结论语言：

```text
这家公司不是因为当期便宜而有吸引力，而是因为 X 子工作流和 Y 子工作流显示利润中枢可能上移；但 Z 子工作流显示当前估值已经计入部分预期，因此后续股价弹性取决于 A/B/C 是否继续兑现。
```
