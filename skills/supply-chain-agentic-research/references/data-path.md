# 数据路径与证据优先级

## 目标

本 skill 不直接替代 `$tushare`、`$wencai-query`、公告 PDF 处理、客户 capex 或海关/出货数据能力。它负责确定证据链、调用顺序和口径解释。

## 证据优先级

1. 官方公告原文：年报、季报、业绩预告/快报、订单、合同、募投、定增、可转债、问询回复、业绩说明会。
2. 客户和行业一手资料：客户财报、资本开支指引、电话会、供应链公开数据、产品平台路线、行业协会或权威机构数据。
3. `$tushare` skill：股价、市值、PE/PB、财务三表、财务指标、现金流、历史行情、行业和股本。
4. `$wencai-query` skill：题材概念、市场排名、同行筛选、阶段涨幅、事件性条件和快速交叉验证。
5. 公司官网、投资者关系、交易所互动、可靠新闻和券商纪要：用于补充客户、订单、产品、产能和产业趋势。
6. 海关、出货、排产和产业链跟踪数据：用于验证订单是否真实放量。若来源不是官方，必须说明可靠性和口径限制。

公告原文与结构化数据冲突时，以公告原文为准，并说明差异口径。行情和估值必须注明查询日期。

## 单公司深度研究

默认路径：

```text
1. 形成主线假设
2. 读最新年报/季报/业绩预告/重大事项公告
3. 调用 `$tushare` skill 拉股价、市值、估值、财务和现金流
4. 调用 `$wencai-query` skill 查概念、同行和市场位置
5. 核验客户 capex、产业周期和产品平台迭代
6. 分发给周期、地位、订单、利润中枢与估值、反方审查角色
7. 做利润桥和估值反推
8. 写投资人可读报告
```

## 快速判断

当用户只要快速看一只股票，可以使用本 skill 的轻量模式，也可以改用原 `$supply-chain-platform-growth`。若使用本 skill，必须说明未启用完整 agentic 流程。

轻量模式至少覆盖：

- 产业周期。
- 供应链地位。
- 订单出货。
- 当前估值是否透支。

不能把轻量模式输出包装成完整深度报告。

## 横向比较或选股

当用户要求“找一批”“横向比较”“谁更强”：

```text
`$wencai-query` skill：先筛候选池、题材口径和市场排名
`$tushare` skill：补财务、估值、现金流和行情指标
公告原文：只抽 Top 候选或争议公司的关键公告
产业数据：按环节比较客户 capex、价值量、订单和竞争扩产
```

不要对所有候选公司逐一读年报，先用数据缩小范围。

## 订单/扩产/业绩驱动

当研究重点是订单、扩产、业绩预告、海外基地、客户验证：

```text
公告原文优先
订单/排产/出货/海关数据辅助
`$tushare` skill 补市值、行情、现金流和财务承受能力
`$wencai-query` skill 补市场反应和同类标的
```

必须区分“事实已公告”“公司预计”“产业链推断”“市场情绪”。

## 子模块对应数据

| 模块 | 优先数据 | 辅助数据 |
| --- | --- | --- |
| 产业周期 | 客户 capex、客户财报、行业出货、产品路线 | `$wencai-query` 同行和概念、可靠新闻 |
| 供应链地位 | 年报客户/产品、认证、官网、投资者关系 | 同行资料、竞争扩产、毛利率对比 |
| 订单出货 | 订单公告、量产、出货、海关、排产 | 财报收入、存货、应收、合同负债 |
| 承接动作 | 扩产、募投、设备、海外基地、研发投入 | `$tushare` 现金流、负债、在建工程 |
| 利润桥 | 财报分产品、毛利率、订单、产能和成本 | 行业毛利率、同行净利率、募投效益 |
| 远期估值 | `$tushare` 市值/估值/行情、利润桥 | 同行估值、历史估值、市场阶段 |

## 关键数据口径

必须注明：

- 股价、市值和估值的查询日期。
- 财报或业绩预告的报告期和披露日。
- 客户 capex 的来源、币种、会计年度和是否为公司相关业务。
- 订单金额是否含税、是否框架协议、是否已量产或收入确认。
- 海关/出货数据的统计口径、商品编码、地区和时间范围。
- 利润假设是事实、合理推断还是情景假设。

## 证据强度分级

| 证据等级 | 例子 | 用法 |
| --- | --- | --- |
| 强证据 | 公告订单、财报利润、客户 capex 指引、明确量产出货 | 可支撑主结论 |
| 中证据 | 扩产、设备、认证、客户合作、行业出货趋势 | 可支撑推断，但需后续验证 |
| 弱证据 | 传闻、概念、未量化表述、笼统进入供应链 | 只能作为线索 |

## 输出口径

每份报告至少说明：

- 使用了哪些 skill 路径：公告、`$tushare`、`$wencai-query`、客户 capex、海关/出货、网页核验。
- 最新数据日期和报告期。
- 哪些结论来自公告事实，哪些来自推算或情景假设。
- 哪些证据提高确信度，哪些证据降低确信度。

## 兜底原则

如果客户 capex、订单或海关数据缺失，不能强行得出“周期继续”和“订单放量”的强结论。应降级为“正在验证”或“证据不足”，并列出后续必须跟踪的数据。
